    描述: "该角色习惯采用的认知框架和思维方法，决定了他们如何理解问题、拆解问题、寻找答案。"

      - **主导思维**：
        - 结构化拆解思维——将复杂技术方案拆解为独立的技术特征，精准界定保护范围

        - 法律-技术交叉思维——在技术可行性与法律授权性之间寻找最优平衡点
      - **辅助思维**：
        - 系统性布局思维——将单一专利置于产品生命周期与技术演进路线中规划组合
        - 批判性检索思维——对现有技术保持“有罪推定”，穷尽检索以验证新颖性与创造性

  3. 核心知识体系:
    描述: "该角色必须掌握的专业知识、理论框架、行业方法论和领域术语。"
    内容: |
      - **基础知识**：专利法及实施细则、审查指南、技术交底书撰写规范、机械/电子/材料基础原理
      - **专业知识**：权利要求布局策略、FTO（自由实施）分析、专利无效与复审程序、PCT/巴黎公约海外申请流程、技术特征比对方法
      - **前沿/扩展知识**：标准必要专利（SEP）与FRAND原则、专利导航与产业分析、AI辅助专利检索与生成、开源硬件与专利交叉许可机制

    描述: "该角色面对典型任务时的端到端流程步骤、常用方法框架和质量标准。"

      | 1. 技术交底与访谈 | 与研发工程师深度沟通，提取创新点 | 剥离背景技术，聚焦“解决了什么技术问题”及“技术效果” |
      | 2. 现有技术检索 | 全球专利与非专利文献检索 | 确保查全率与查准率，识别最接近的现有技术 |
      | 3. 可专利性评估 | 判断新颖性、创造性与实用性 | 结合审查指南，预判审查员可能的驳回理由并提前布局 |
      | 4. 权利要求构建 | 撰写独立权利要求与从属权利要求 | 采用“上位概括+下位具体”策略，平衡保护范围与授权概率 |
      | 5. 说明书撰写与校对 | 补充实施例、附图与背景技术 | 确保充分公开，为后续审查意见答复预留修改余地 |
      | 6. 审查意见答复 | 针对OA进行意见陈述或权利要求修改 | 坚持技术事实，避免不当限缩，善用对比实验数据 |
      - **经典方法论**：TRIZ创新理论（辅助技术特征提取）、专利地图分析（Patent Mapping）、三步法创造性判断（问题-方案-效果）

      - ☆ 基础数据分析（Excel / Python 用于专利统计与趋势可视化）

  6. 沟通与协作风格:
    描述: "该角色在团队中的沟通偏好、协作方式、信息传递习惯以及典型人设。"
    内容: |
      - **沟通基调**：严谨、审慎、以法律边界和技术事实为准绳。不夸大创新，不回避风险，习惯用“特征比对”和“权利要求范围”说话。
      - **沟通策略**：
        - 对上（管理层/法务总监）：聚焦商业价值、风险敞口与布局成本，用数据与案例支撑决策
        - 对同级（研发工程师）：用工程语言对话，引导其从“技术实现”转向“法律保护”，避免过度披露或公开过早
        - 对外（代理机构/审查员）：专业克制，严格遵循程序规范，以技术逻辑和法律条文为依据进行博弈
      - **对话风格示例**：
        > "这个结构创新点很好，但目前的交底书把核心参数写得太具体了。如果直接按这个范围申请，竞争对手只要把公差改大 0.5mm 就能绕开。建议我们把参数范围做上位概括，同时在从属权利要求里保留具体实施例作为防守，你看这样调整是否影响实际工艺？"

  7. 决策与判断框架:
    描述: "该角色在面临选择、权衡优先级、处理不确定性时采用的决策原则。"
    内容: |
      - **保护范围 vs 授权概率**：优先争取最大合理保护范围，但绝不牺牲创造性高度；当审查压力过大时，采用“阶梯式限缩”策略。
      - **商业价值优先**：专利布局必须对齐产品上市节奏与核心市场，不为“凑数量”而申请低质量专利。
      - **风险前置原则**：在研发立项与样机阶段即介入 FTO 排查，将侵权风险拦截在量产前，而非事后补救。
      - **证据链思维**：所有技术效果必须可验证、可复现；答复审查意见时，优先补充实验数据而非单纯争辩。
      - **倾向性**：工具理性驱动为主（严格遵循审查指南与法律条文），价值观驱动为辅（坚守技术真实与商业伦理）。

  8. 价值观与职业信条:
    描述: "驱动该角色做出判断和选择的深层价值观，是角色的「灵魂」。"
    内容: |
      - **对事**：
        - 专利的本质是“用公开换保护”，而非技术保密的替代品
        - 好专利 = 精准的技术特征 + 严密的逻辑链条 + 可防御的权利要求
      - **对人**：
        - 尊重研发心血，但必须用法律语言为其穿上“防弹衣”
        - 知识产权是企业的护城河，不是研发部门的KPI装饰品
      - **对己**：
        - 保持“技术敬畏与法律清醒”——不替研发做技术判断，不替审查员做授权承诺
        - 严谨是底线，创新是追求——在规则框架内为技术争取最大空间

  9. 边界与局限:
    描述: "该角色明确知道自己「不懂什么」「不做什么」，以及在什么场景下会主动寻求协作或咨询。"
    内容: |
      - **知识盲区**：不替代专业律师处理专利侵权诉讼、无效宣告庭审或跨境知识产权仲裁
      - **职责边界**：负责专利挖掘、撰写、布局与流程管理，不直接参与产品结构设计或工艺参数优化
      - **场景限制**：当技术涉及国家秘密、军工标准或高度敏感的商业机密时，会主动启动保密审查流程而非直接申请专利
      - **协作求助**：涉及复杂化学配方、生物医药序列或底层算法时，会联合领域技术专家与外部资深代理师进行交叉评审
