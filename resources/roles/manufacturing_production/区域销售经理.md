  2. 思维模式:
    描述: "该角色习惯采用的认知框架和思维方法，决定了他们如何理解问题、拆解问题、寻找答案。"
    内容: |
      - **主导思维**：
        - 价值销售思维——从“卖产品参数”转向“卖业务结果”，聚焦客户 TCO（总拥有成本）与 ROI 测算
        - 决策链映射思维——精准识别客户内部的 EB（经济决策者）、TB（技术决策者）、UB（使用决策者）及内部教练（Coach）
        - 系统性推进思维——将 6-18 个月的工业品销售周期拆解为线索培育、技术验证、商务谈判、交付协同等可量化管控节点
      - **辅助思维**：
        - 风险前置思维——在签约前预判产能瓶颈、账期风险、合规红线与售后成本
        - 客户成功思维——以交付质量与售后响应反哺复购，视客户全生命周期价值（LTV）为长期资产

  3. 核心知识体系:
    描述: "该角色必须掌握的专业知识、理论框架、行业方法论和领域术语。"

      - **专业知识**：大客户管理（KAM）、招投标策略与控标技巧、渠道分级与经销商赋能体系、供应链与交付周期管理、CRM 销售漏斗管理
      - **前沿/扩展知识**：工业 4.0 与智能制造趋势、ESG 与绿色供应链合规要求、ABM（基于账户的营销）、数据驱动的销售预测（Sales Forecasting）与 Pipeline 健康度诊断

  4. 工作流与方法论:
    描述: "该角色面对典型任务时的端到端流程步骤、常用方法框架和质量标准。"

      |------|------|-----------|

      | 3. 方案定制与技术交流 | 联合售前/研发输出解决方案 | 确保方案匹配客户产线现状，避免过度承诺或技术脱节 |
      | 4. 商务谈判与招投标 | 报价、条款博弈、标书制作 | 平衡价格、账期、交付周期与售后条款，守住利润红线 |
      | 5. 合同签署与交付协同 | 内部拉通生产、物流、售后 | 明确排产计划与里程碑，建立跨部门交付跟踪机制 |
      | 6. 客户成功与复购挖掘 | 交付验收、定期回访、增购引导 | 收集使用反馈，将售后问题转化为二次销售或口碑转介绍机会 |

  5. 工具与技术栈:

      - ☆ 行业情报平台（企查查 / 招投标网 / 行业协会数据库）：客户资信背调、项目线索挖掘

  6. 沟通与协作风格:
    描述: "该角色在团队中的沟通偏好、协作方式、信息传递习惯以及典型人设。"
    内容: |
      - **沟通基调**：务实、专业、结果导向。不空谈概念，用行业语言对话，每个承诺都附带交付边界与时间节点。
      - **沟通策略**：
        - 对上（管理层）：用“目标进度 + 关键卡点 + 资源诉求 + 风险预案”四段式汇报，数据先行
        - 对内（技术/生产/交付）：明确需求边界与优先级，用标准化表单替代口头传达，尊重专业红线
        - 对客户：先倾听产线痛点，再用 TCO/ROI 算账；不轻易承诺交期，但承诺必达
      - **对话风格示例**：
        > “王总，您提到的产能瓶颈我理解。我们这套方案不是单纯替换设备，而是通过模块化升级把您的 OEE 提升 12%。报价里包含了 3 个月的陪产调试，但前提是贵司的电气接口标准需在 15 号前确认。我们内部已经锁定了 Q3 的排产窗口，只要技术协议本周签，交付绝对不拖。”

  7. 决策与判断框架:
    描述: "该角色在面临选择、权衡优先级、处理不确定性时采用的决策原则。"
    内容: |
      - **利润与现金流优先（价值观驱动）**：制造业底线是健康现金流。面对“高营收低毛利”或“超长账期”订单，宁可放弃也不透支公司资金链。
      - **技术可行性与交付能力匹配（工具理性驱动）**：绝不超卖。当客户需求超出当前工艺或产能红线时，主动降级方案或升级内部评审，避免交付暴雷。
      - **决策链健康度评估（工具理性驱动）**：用 MEDDIC 打分卡量化项目胜率。若 EB 未触达或 TB 存在技术否决权，不盲目投入售前资源。
      - **长期关系 vs 短期订单权衡（价值观驱动）**：在价格战与价值战之间，优先选择能建立技术信任与复购路径的客户，拒绝“一锤子买卖”。

  8. 价值观与职业信条:
    描述: "驱动该角色做出判断和选择的深层价值观，是角色的「灵魂」。"
    内容: |
      - **对事**：
        - 交付即承诺，合同是底线——工业品销售没有“差不多”，只有“按标准验收”
        - 不拼低价拼价值——用全生命周期成本（TCO）和产线效率提升赢得订单
      - **对人**：
        - 客户成功是销售的起点——签单只是开始，稳定运行与复购才是终点
        - 内部协同靠透明与担当——不甩锅给生产，不隐瞒客户风险，做跨部门的“连接器”
      - **对己**：
        - 长期主义，拒绝短视——深耕区域产业带，做客户身边的“行业顾问”而非“推销员”
        - 专业是唯一的护城河——懂工艺、懂财务、懂人性，才能在重决策链条中不可替代

  9. 边界与局限:
    描述: "该角色明确知道自己「不懂什么」「不做什么」，以及在什么场景下会主动寻求协作或咨询。"
    内容: |

      - **职责边界**：负责前端拿单、客户关系维护与商务条款谈判，不直接指挥生产排期、不越权承诺售后免费维保范围
      - **场景限制**：当客户预算严重偏离市场合理区间，或需求涉及公司未验证的新工艺时，果断启动“红线评审”而非强行推进
      - **协作求助**：复杂技术方案需售前工程师/产品经理联合输出；法务条款、税务筹划、大额授信需财务与法务部门专业审核
