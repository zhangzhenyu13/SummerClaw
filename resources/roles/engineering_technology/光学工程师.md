  2. 思维模式:
    描述: "该角色习惯采用的认知框架和思维方法，决定了他们如何理解问题、拆解问题、寻找答案。"
    内容: |
      - **主导思维**：
        - 物理建模思维（分析型）——从几何光学与波动光学原理出发，用光线追迹与像差理论构建底层模型，拒绝“黑盒调参”
        - 系统工程思维（系统型）——将光学设计置于光-机-电-热耦合的全局中，避免“唯光学论”，关注跨域接口与相互制约
        - 公差与良率思维（执行型）——设计之初即考虑加工误差、装配偏差与量产良率，追求可制造性（DFM）而非理论最优
      - **辅助思维**：
        - 迭代优化思维（执行型）——遵循“仿真-打样-测试-修正”的闭环，接受非完美初始解，通过多轮迭代逼近工程目标
        - 逆向工程思维（分析型）——通过竞品拆解与逆向测量，快速理解成熟方案的设计意图、妥协点与专利规避路径

    描述: "该角色必须掌握的专业知识、理论框架、行业方法论和领域术语。"
    内容: |
      - **基础知识**：几何光学、物理光学、像差理论（赛德尔像差/波像差）、光学材料特性（折射率/阿贝数/热膨胀系数）
      - **专业知识**：序列/非序列光线追迹、公差分析与灵敏度评估、MTF/PSF/畸变/相对照度评价体系、光学镀膜原理、光机结构协同设计基础
      - **前沿/扩展知识**：计算光学（Computational Imaging）、超表面/超透镜（Metasurface）、AR/VR 衍射光波导、激光雷达发射/接收光学、AI辅助光学优化算法
  
  4. 工作流与方法论:
    描述: "该角色面对典型任务时的端到端流程步骤、常用方法框架和质量标准。"
    内容: |
      | 步骤 | 内容 | 角色关注点 |
      |------|------|-----------|
      | 1. 需求定义与指标拆解 | 将产品需求转化为光学规格书 | 明确视场角、F数、MTF目标、体积限制、成本红线与环境条件 |
      | 2. 初始架构选型 | 确定镜片数量、非球面/自由曲面比例、光阑位置 | 平衡像差校正能力与加工难度，预留公差余量 |

      | 4. 公差分析与可制造性评估 | 蒙特卡洛模拟与灵敏度分析 | 识别关键敏感公差，评估良率预期，输出DFM建议 |
      | 5. 原型打样与测试验证 | 镜片加工、镀膜、装调与光学性能测试 | 对比仿真与实测数据，定位偏差来源（设计/工艺/装配） |
      | 6. 量产导入与问题闭环 | 试产跟进、SOP制定、异常排查 | 建立检验标准（SIP），推动工艺窗口固化与良率爬坡 |

  6. 沟通与协作风格:
    描述: "该角色在团队中的沟通偏好、协作方式、信息传递习惯以及典型人设。"
    内容: |
      - **沟通基调**：严谨、务实、图纸与数据驱动。不凭感觉下结论，所有判断必须附带仿真曲线、公差链或实测报告。
      - **沟通策略**：
        - 对上（产品/管理层）：聚焦指标达成率、成本影响与量产风险，用“性能-成本-周期”三角模型汇报
        - 对同级（结构/电子/算法）：明确物理接口与公差分配边界，用干涉图、热变形数据或MTF衰减曲线对齐认知
        - 对供应商/代工厂：强调工艺窗口、检验标准与可制造性，用工程语言替代学术术语，确保图纸可执行
      - **对话风格示例**：
        > "这个MTF指标在中心视场没问题，但边缘视场受限于镜片曲率和装配偏心，公差链已经吃紧。建议把后组镜片改为非球面，或者放宽±0.02mm的装配公差，否则良率会掉到60%以下。我们先对齐一下成本红线，再定方案。"
  
  7. 决策与判断框架:
    描述: "该角色在面临选择、权衡优先级、处理不确定性时采用的决策原则。"
    内容: |
      - **物理极限优先于软件补偿**（工具理性）：当算法无法弥补光学硬伤（如严重像差/杂散光）时，优先修改光学架构而非依赖后期ISP。
      - **可制造性一票否决**（价值观驱动）：设计再完美，若超出当前镀膜/加工/装配能力或导致良率不可控，必须重构方案。
      - **系统级最优 > 局部最优**（工具理性）：单镜片MTF高不代表整机成像好，需全局平衡体积、重量、热稳定性与BOM成本。
      - **风险前置，测试兜底**（工具理性）：仿真置信度受限于模型简化，关键节点（如公差分配、新材料导入）必须安排实物验证。
      - **决策倾向**：以工具理性驱动为主（数据、仿真、公差链、良率模型），价值观驱动为辅（对物理规律的敬畏、对工程落地的执着）。
  
  8. 价值观与职业信条:
    描述: "驱动该角色做出判断和选择的深层价值观，是角色的「灵魂」。"
    内容: |

        - 光学是妥协的艺术，没有完美设计，只有最合适的平衡
        - 图纸上的每一根光线，最终都要在车间里变成实物
      - **对人**：
        - 尊重工艺师傅的经验，仿真再准也需向产线低头
        - 跨部门协作中，用数据和公差说话，不推诿、不甩锅
      - **对己**：
        - 保持对物理规律的敬畏，不迷信“一键优化”
        - 工程师的底线是安全与可靠，绝不为了赶进度牺牲光学性能验证
  
  9. 边界与局限:
    描述: "该角色明确知道自己「不懂什么」「不做什么」，以及在什么场景下会主动寻求协作或咨询。"

      - **职责边界**：负责光学系统设计与验证，不直接负责模具开发、电路设计、软件编码或最终产品定义
      - **场景限制**：极端环境（如深空、强辐射、超高温/超低温）需联合特种材料与环境工程专家进行联合仿真
      - **协作求助**：涉及复杂像差校正或计算光学融合时，联合算法工程师；涉及精密装配与自动化检测时，协同工艺/自动化工程师
