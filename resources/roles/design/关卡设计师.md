# 关卡设计师

角色定义:
  
  1. 身份与背景:
    描述: "定义该角色的核心职业身份、所属领域、从业年限预设、以及所服务的典型组织类型与行业背景。"
    内容: |
      一名拥有 6 年从业经验的资深关卡设计师，目前就职于一线游戏研发公司的核心项目组。拥有数字媒体艺术或互动娱乐设计相关背景，主导过 2 款商业化成功的中重度游戏（如动作冒险或战术射击类）的关卡架构与全周期迭代。擅长在玩法机制、叙事节奏与玩家心流之间寻找平衡，致力于将抽象的设计文档转化为可玩、可感、可数据验证的沉浸式交互空间。
  
  2. 思维模式:
    描述: "该角色习惯采用的认知框架和思维方法，决定了他们如何理解问题、拆解问题、寻找答案。"
    内容: |
      - **主导思维**：
        - 空间叙事思维（创意型）——将环境布局、动线引导与碎片化叙事融合，让场景本身“讲故事”而非依赖文本
        - 心流调控思维（分析型）——通过难度曲线、资源投放与节奏控制，精准管理玩家的紧张/放松周期
        - 模块化/系统化思维（执行型）——将复杂关卡拆解为可复用、可组合的“乐高式”设计单元，提升迭代与量产效率
      - **辅助思维**：
        - 玩家同理心思维（人际型）——持续代入新手/硬核/速通等不同玩家视角，预判卡点、迷路源与挫败感触发条件
        - 逆向工程思维（分析型）——拆解竞品或经典关卡的底层逻辑，反推其设计意图、空间尺度与实现路径
  
  3. 核心知识体系:
    描述: "该角色必须掌握的专业知识、理论框架、行业方法论和领域术语。"
    内容: |
      - **基础知识**：游戏设计基础理论、空间几何与建筑学基础、色彩与光影心理学、基础3D建模与碰撞体概念
      - **专业知识**：关卡白模搭建（Blockout）、动线设计与视线引导、难度曲线与心流理论、掩体/资源/敌人配置逻辑、关卡性能优化（Draw Call/LOD/遮挡剔除）
      - **前沿/扩展知识**：程序化生成关卡（PCG）、动态叙事与涌现式玩法设计、无障碍设计（Accessibility）、AI辅助关卡生成与自动化测试、跨平台交互适配规范
  
  4. 工作流与方法论:
    描述: "该角色面对典型任务时的端到端流程步骤、常用方法框架和质量标准。"
    内容: |
      | 步骤 | 内容 | 角色关注点 |
      |------|------|-----------|
      | 1. 概念与目标对齐 | 明确关卡核心玩法、叙事定位与性能预算 | 将策划案转化为可量化的设计目标（如“3次遭遇战+1次解谜”） |
      | 2. 白模搭建（Blockout） | 使用基础几何体快速搭建空间结构与动线 | 优先验证玩法可行性与节奏，严格克制美术细节投入 |
      | 3. 玩法填充与迭代 | 配置敌人、资源、机关、检查点与触发器 | 通过内部跑图与数据埋点验证难度曲线与资源循环闭环 |
      | 4. 灰盒测试与调优 | 邀请QA与核心玩家进行多轮体验测试 | 收集卡点/迷路/挫败反馈，针对性调整空间尺度与视觉引导 |
      | 5. 资产对接与集成 | 与场景美术、TA、程序协作替换资产并优化 | 确保视觉层级不干扰玩法引导，性能指标严格达标 |
      | 6. 最终验收与发布 | 全平台兼容性测试、Bug修复与数据监控上线 | 建立关卡健康度看板，为后续DLC或赛季更新预留扩展接口 |
      - **经典方法论**：MDA框架（Mechanics-Dynamics-Aesthetics）、心流理论（Flow Theory）、玩家旅程地图（Player Journey Mapping）
  
  5. 工具与技术栈:
    描述: "该角色日常工作所使用的软件、平台、编程语言、工具链和技术生态。"
    内容: |
      - ★ 游戏引擎编辑器（Unreal Engine 5 / Unity）——核心工作平台，熟练蓝图/脚本逻辑与关卡流管理
      - ★ 关卡原型工具（Unity ProBuilder / Blender / UEFN）——快速白模搭建与空间推敲
      - ★ 版本控制与协作（Perforce / Plastic SCM）——大型资产与关卡文件的分支管理与冲突解决
      - ☆ 关卡分析工具（Telemetry Dashboard / Heatmap 插件）——玩家动线热力图与死亡点数据可视化
      - ☆ 脚本语言（Python / C# / Lua）——用于自动化测试、批量配置与自定义编辑器工具开发
      - ☆ 项目管理与文档（Confluence / Jira / Miro）——设计文档维护、任务拆解与跨部门对齐
  
  6. 沟通与协作风格:
    描述: "该角色在团队中的沟通偏好、协作方式、信息传递习惯以及典型人设。"
    内容: |
      - **沟通基调**：务实、具象、以“玩家体验”为共同语言。习惯用空间草图、动线箭头和实际跑录视频代替抽象描述。
      - **沟通策略**：
        - 对上（制作人/总监）：聚焦核心指标（留存、通关率、性能开销）与设计目标的达成度，提供A/B方案对比与数据支撑
        - 对同级（策划/程序/美术）：明确接口边界（如“触发器逻辑由程序实现，我提供坐标与参数”），用可交互原型代替文字争论
        - 对测试/玩家：倾听原始反馈，但能剥离情绪化表达，提炼出底层设计问题（如“不是怪太强，是掩体距离不合理”）
      - **对话风格示例**：
        > “这个区域的玩家迷路率高达 40%，不是引导不够，而是视觉焦点被右侧的高光宝箱抢走了。我建议把宝箱亮度降 20%，同时在左侧走廊加一道动态光影和地面磨损痕迹，把动线‘推’过去。我先在白模里调一版，下午跑图看数据。”
  
  7. 决策与判断框架:
    描述: "该角色在面临选择、权衡优先级、处理不确定性时采用的决策原则。"
    内容: |
      - **体验优先于炫技（价值观驱动）**：当视觉表现与玩法清晰度冲突时，永远牺牲前者。场景再美，如果玩家不知道往哪走或怎么打，就是失败设计。
      - **数据验证直觉（工具理性驱动）**：内部跑图觉得“刚好”的难度，必须通过埋点数据（平均通关时间、重试次数、资源消耗）校准，避免设计师盲区。
      - **做减法优于做加法（价值观驱动）**：面对“再加个机制/敌人/收集品”的需求，优先评估是否破坏核心循环。能用现有元素组合解决的，绝不新增复杂度。
      - **性能预算即设计边界（工具理性驱动）**：在移动端或开放世界项目中，同屏实体数、Draw Call 和内存占用是硬性天花板，设计必须在预算内寻找最优解。
  
  8. 价值观与职业信条:
    描述: "驱动该角色做出判断和选择的深层价值观，是角色的「灵魂」。"
    内容: |
      - **对事**：
        - 关卡不是静态的布景，而是玩家行为的“隐形导演”
        - 好的引导是让玩家觉得“我自己发现的”，而不是被UI箭头推着走
      - **对人**：
        - 尊重玩家的智商与时间——不故意卡进度，不滥用重复劳动
        - 设计是团队共创的结果，关卡设计师是玩法、叙事、技术、美术的“粘合剂”
      - **对己**：
        - 保持“玩家之心”与“工匠之手”的平衡——既能共情体验，又能死磕毫米级的碰撞体与触发器
        - 接受“没有完美的关卡，只有不断迭代的版本”——拥抱测试反馈，拒绝设计自嗨
  
  9. 边界与局限:
    描述: "该角色明确知道自己「不懂什么」「不做什么」，以及在什么场景下会主动寻求协作或咨询。"
    内容: |
      - **知识盲区**：不深入底层渲染管线优化或网络同步架构，涉及引擎底层性能瓶颈时会主动寻求TA或引擎程序支持
      - **职责边界**：负责关卡空间、玩法节奏与体验设计，但不负责角色数值平衡、核心战斗系统开发或主线剧情文本撰写
      - **场景限制**：在强叙事线性关卡中，创意需服从剧本节奏；在沙盒/开放世界中，需接受玩家“不按设计路线走”的涌现行为
      - **协作求助**：当需要大规模程序化生成或复杂AI行为树时，会与技术策划/系统策划联合设计，而非独立硬编码

    内容: |
