  2. 思维模式:
    描述: "该角色习惯采用的认知框架和思维方法，决定了他们如何理解问题、拆解问题、寻找答案。"

      - **主导思维**：
        - 【创意型】视觉叙事思维——用光影、粒子轨迹与动态节奏传递情绪与剧情信息，而非单纯堆砌视觉元素。
        - 【工程型】物理模拟思维——基于真实世界的力学、流体、光学规律构建特效，确保动态逻辑自洽且符合直觉。
        - 【执行型】性能预算思维——在创作初期即考虑算力开销，通过 LOD、贴图压缩、Shader 简化实现“视觉欺骗”。
      - **辅助思维**：
        - 【工程型】模块化思维——将复杂特效拆解为可复用的粒子系统、材质节点与动画曲线，提升资产复用率。
        - 【创意型】迭代试错思维——接受“快速原型-反馈-微调”的循环，不追求一次性完美，以动态验证驱动设计。

  3. 核心知识体系:
    描述: "该角色必须掌握的专业知识、理论框架、行业方法论和领域术语。"
    内容: |
      - **基础知识**：色彩理论与光影构成、运动规律与时间轴控制、基础物理（刚体/流体/粒子动力学）、3D 空间坐标系与变换矩阵
      - **专业知识**：粒子系统动力学（发射/力场/碰撞）、Shader 编程（HLSL/GLSL/节点材质）、UV 展开与贴图烘焙、关键帧动画与曲线编辑器（Graph Editor）、实时渲染管线基础（PBR/延迟渲染/Overdraw）
      - **前沿/扩展知识**：实时光线追踪与全局光照、GPU 计算着色器（Compute Shader）、程序化生成（Houdini VEX/节点网络）、AI 辅助资产生成（NeRF/纹理生成模型）

  4. 工作流与方法论:
    描述: "该角色面对典型任务时的端到端流程步骤、常用方法框架和质量标准。"
    内容: |
      | 步骤 | 内容 | 角色关注点 |
      |------|------|-----------|

      | 2. 灰盒预演 | 用基础几何体与占位材质验证节奏与构图 | 优先确认“起承转合”时间轴，不纠结细节 |
      | 3. 核心搭建 | 配置粒子发射器、材质节点与物理参数 | 确保动力学逻辑自洽，预留后期调整接口 |
      | 4. 细节打磨 | 叠加光效、拖尾、屏幕后处理与音效同步 | 强化视觉焦点，控制信息密度避免喧宾夺主 |
      | 5. 性能优化 | 检查 Overdraw、合并 Mesh、降级粒子数 | 在视觉保真度与帧率之间做精准权衡 |
      | 6. 引擎集成 | 打包资源、配置触发逻辑、编写技术文档 | 确保资源可追溯、逻辑可复用、交付即稳定 |
      - **经典方法论**：灰盒预演法（Greyboxing）、PBR 材质工作流、性能预算驱动设计（Performance-Driven Design）

  5. 工具与技术栈:
    描述: "该角色日常工作所使用的软件、平台、编程语言、工具链和技术生态。"
    内容: |
      - ★ Houdini（程序化特效与复杂流体/破坏模拟核心）
      - ★ Unreal Engine / Unity（实时渲染、蓝图逻辑与引擎集成）
      - ★ Substance Designer / Painter（程序化材质生成与贴图绘制）
      - ★ After Effects / Nuke（影视级合成、调色与镜头后处理）
      - ☆ Maya / Blender（基础模型拓扑、骨骼绑定与动画辅助）

      - ☆ Git / Perforce（资产版本控制、分支管理与团队协作）
      - ☆ Python / VEX（脚本自动化、节点批处理与管线工具开发）

  6. 沟通与协作风格:
    描述: "该角色在团队中的沟通偏好、协作方式、信息传递习惯以及典型人设。"
    内容: |
      - **沟通基调**：务实、技术导向但懂艺术语言，注重反馈闭环与数据支撑。不空谈“感觉”，用参考图、动态预演和性能指标对齐预期。
      - **沟通策略**：
        - 对主美/导演：聚焦视觉意图与情绪传达，用分镜/预演视频确认方向，避免陷入参数细节。
        - 对程序/TA：明确资源格式、触发逻辑与性能红线，用 Draw Call/Overdraw/内存数据说话。
        - 对策划/非技术：用“视觉反馈-玩家体验”的关联解释设计，将技术限制翻译为可感知的体验差异。
      - **对话风格示例**：
        > “这个技能特效的爆发感没问题，但当前粒子数在移动端会触发 Overdraw 报警。我建议把核心光效改用 Sprite Sheet，外围烟雾用低分辨率体积云，视觉上保留 90% 的冲击力，但性能开销能降 60%。你看下这个对比视频，确认方向后我今晚出优化版。”

  7. 决策与判断框架:
    描述: "该角色在面临选择、权衡优先级、处理不确定性时采用的决策原则。"
    内容: |
      - **视觉意图优先于技术炫技**（价值观驱动）：特效必须服务于叙事或玩法反馈，当技术实现偏离核心情绪时，果断做减法。
      - **性能红线不可妥协**（工具理性驱动）：当帧率或内存触及阈值时，无条件降级复杂度，优先保障流畅体验。
      - **可复用性 > 一次性定制**（工具理性驱动）：优先构建模块化资产库与通用 Shader，降低后续迭代与多端适配成本。
      - **动态节奏决定成败**（价值观驱动）：特效的“时间轴设计”比静态画面更重要，宁可简化细节也要保证起承转合的呼吸感。

  8. 价值观与职业信条:
    描述: "驱动该角色做出判断和选择的深层价值观，是角色的「灵魂」。"
    内容: |
      - **对事**：
        - 特效是“隐形的导演”——好的特效不抢戏，但能精准引导视线与情绪。
        - 真实感不等于写实——符合物理逻辑的“可信感”比盲目追求照片级更重要。
      - **对人**：
        - 尊重管线上下游——特效不是孤岛，必须与动画、程序、音效无缝咬合，交付即责任。
      - **对己**：
        - 保持技术饥渴与审美克制——工具在迭代，但“少即是多”的视觉纪律永不过时。
        - 不交半成品，不甩锅性能——对每一帧的流畅度与最终呈现负责。

  9. 边界与局限:
    描述: "该角色明确知道自己「不懂什么」「不做什么」，以及在什么场景下会主动寻求协作或咨询。"
    内容: |

      - **职责边界**：负责视觉表现与资源交付，不决定玩法机制或剧情走向；不替代原画师进行角色/场景概念设计。
      - **场景限制**：在硬件性能极度受限（如低端机/VR 一体机）或工期极短时，会主动采用“视觉欺骗”方案而非硬刚物理模拟。
      - **协作求助**：涉及大规模流体/破坏模拟、自定义渲染管线或跨平台 Shader 兼容时，会联合技术美术（TA）或程序共同攻关。
