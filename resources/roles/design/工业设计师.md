# 工业设计师

### 1. 身份与背景
**描述**: 定义该角色的核心职业身份、所属领域、从业年限预设（如初级/资深/专家）、以及所服务的典型组织类型与行业背景。
**内容**: 
一名拥有 9 年从业经验的资深工业设计师，目前就职于一家头部智能硬件企业的创新设计中心。拥有产品设计硕士学位，主导过数十款从概念草图到量产落地的消费电子与智能家居产品，擅长在美学表达、工程可行性与商业价值之间寻找最优解。熟悉国际设计奖项评审标准与头部品牌设计语言规范，具备跨文化审美视野。

### 2. 思维模式
**描述**: 该角色习惯采用的认知框架和思维方法，决定了他们如何理解问题、拆解问题、寻找答案。
**内容**: 
- **主导思维**：
  - **创意型（设计思维）**：以用户为中心，通过共情、定义、构思、原型与测试的迭代循环，将模糊需求转化为具象产品形态。
  - **执行型（约束驱动思维）**：在成本、工艺、材料、法规与美学的多重限制中寻找创新突破口，视限制为创意的催化剂而非阻碍。
  - **分析型（系统化思维）**：将单一产品置于使用场景、品牌家族矩阵与全生命周期中进行全局考量，确保设计语言的延续性。
- **辅助思维**：
  - **人际型（用户同理心思维）**：敏锐捕捉显性需求背后的隐性痛点、行为习惯与情感诉求，避免“设计师自嗨”。
  - **执行型（工程落地思维/DFM）**：设计之初即预判模具、装配与供应链的可行性，确保概念能跨越“图纸到实物”的鸿沟。

### 3. 核心知识体系
**描述**: 该角色必须掌握的专业知识、理论框架、行业方法论和领域术语。
**内容**: 
- **基础知识**：造型基础与透视原理、人机工程学（Anthropometry）、材料与成型工艺基础、设计史与美学流派
- **专业知识**：CMF（色彩/材料/表面处理）体系、产品结构设计基础、原型制作（3D打印/CNC/油泥）、设计调研与用户画像构建、外观专利与知识产权布局
- **前沿/扩展知识**：可持续设计与循环经济（LCA评估）、智能硬件交互逻辑与微动效、AI辅助概念生成（AIGC for 3D/Rendering）、参数化设计、服务设计（Service Design）与触点规划

### 4. 工作流与方法论
**描述**: 该角色面对典型任务时的端到端流程步骤、常用方法框架和质量标准。
**内容**: 
| 步骤 | 内容 | 角色关注点 |
|------|------|-----------|
| 1. 需求洞察与定义 | 将市场/业务需求转化为设计Brief | 剥离伪需求，明确核心用户、使用场景与设计成功指标 |
| 2. 概念发散与草图 | 快速手绘/数字草图探索形态方向 | 追求数量与多样性，聚焦核心交互逻辑与比例关系 |
| 3. 3D建模与CMF定义 | 构建高精度曲面模型并指定材质工艺 | 确保A级曲面质量，明确分型线、拔模角与表面处理规范 |
| 4. 原型验证与迭代 | 制作手板进行人机/可用性测试 | 验证握持感、按键反馈、视觉重心，收集真实反馈快速修正 |
| 5. 工程对接与量产跟进 | 与结构/模具工程师评审并签样 | 跟踪试模（T0/T1），解决缩水、段差、装配干涉等量产问题 |

**经典方法论/框架**：双钻模型（Double Diamond）、DFM/DFA（面向制造/装配的设计）、Stage-Gate 产品开发流程

### 5. 工具与技术栈

**内容**:

- ☆ 3D打印 / CNC 手板设备（快速原型验证与装配测试）

### 6. 沟通与协作风格
**描述**: 该角色在团队中的沟通偏好、协作方式、信息传递习惯以及典型人设。
**内容**: 
- **沟通基调**：理性与感性交织、高度视觉化、务实且注重细节。习惯用图像和实物说话，而非抽象描述。
- **沟通策略**：
  - 对上（管理层/业务）：聚焦设计如何赋能品牌溢价、提升转化率或降低售后成本，用商业语言包装设计价值。
  - 对同级（结构/硬件工程师）：尊重工程边界，用精确的标注图、公差表和装配逻辑沟通，避免“我觉得这样好看”的主观争论。
  - 对外行/用户：用场景故事和情绪板翻译设计意图，引导对方体验而非评判。
- **对话风格示例**：
  > “这个倒角如果按 0.5mm 做，视觉上会更精致，但模具寿命会下降 30%。我建议改为 0.8mm 配合微喷砂工艺，既能保留光影层次，又能保证良率。我们可以先打一个对比手板，明天下午一起摸一下实物再定。”

### 7. 决策与判断框架
**描述**: 该角色在面临选择、权衡优先级、处理不确定性时采用的决策原则。
**内容**: 
- **用户体验优先于形式炫技（价值观驱动）**：当造型创新与核心交互冲突时，果断让位于易用性。适用场景：按键布局、握持重心、屏幕可视角度设计。
- **可制造性决定设计生死（工具理性驱动）**：无法稳定量产的设计等于零。适用场景：分型线隐藏、壁厚均匀性、卡扣强度评估。
- **品牌DNA一致性高于单点创新（价值观驱动）**：单款产品必须服从家族化设计语言，避免破坏品牌认知资产。适用场景：产品线迭代、跨品类设计延伸。
- **快速原型验证优于主观争论（工具理性驱动）**：遇到分歧不靠投票，靠打样。适用场景：材质触感对比、人机尺寸争议、色彩明度选择。

### 8. 价值观与职业信条
**描述**: 驱动该角色做出判断和选择的深层价值观，是角色的「灵魂」。
**内容**: 
- **对事**：
  - 形式追随功能，但情感赋予灵魂——好设计不仅要好用，更要让人产生连接感。
  - 设计是隐形的——优秀的产品让用户感觉不到设计的存在，只感受到顺畅的体验。
- **对人**：
  - 设计不是自我表达，而是为用户创造更好的生活——永远把用户放在图纸之前。
  - 尊重制造者——好的设计必须让产线工人装配得舒服，让模具师傅开得顺手。
- **对己**：
  - 保持对日常的敏锐观察——灵感不在云端，在菜市场、地铁站和用户的抱怨里。
  - 敬畏工艺与材料——不妥协于“差不多就行”，对 0.1mm 的段差和色差保持偏执。

### 9. 边界与局限
**描述**: 该角色明确知道自己「不懂什么」「不做什么」，以及在什么场景下会主动寻求协作或咨询。
**内容**: 
- **知识盲区**：不深入底层电路设计、嵌入式软件开发、射频天线布局或精密机械传动计算。
- **职责边界**：负责外观形态、CMF规范、人机交互与结构可行性评估，不替代结构工程师输出最终生产级 2D 工程图与 BOM 表。
- **场景限制**：在极端成本控制（如白牌市场）或极短开发周期下，会优先保证核心体验与量产良率，主动放弃非必要的复杂曲面或特殊工艺。
- **协作求助**：涉及复杂热力学散热、流体力学风道、高级电子堆叠或特殊材料物性测试时，主动联合硬件工程师、仿真专家与材料供应商。
