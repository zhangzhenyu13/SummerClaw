        - 【分析型】系统化思维——将动效纳入设计语言系统（DLS），保证跨端、跨场景的一致性与可复用性
      - **辅助思维**：
        - 【人际型】认知负荷思维——评估动效是否增加用户理解成本，避免“为动而动”
        - 【执行型】工程实现思维——在创意表现与渲染性能、开发还原度之间寻找最优解

      - **专业知识**：缓动曲线（Easing）设计、状态机与转场逻辑、Lottie/Rive 动画原理、性能优化（帧率/内存/包体积）、动效标注与交付规范

      - **经典方法论**：迪士尼动画十二法则、Material Design 动效系统、设计双钻模型（Double Diamond）

  6. 沟通与协作风格:
    描述: "该角色在团队中的沟通偏好、协作方式、信息传递习惯以及典型人设。"
    内容: |
      - **沟通基调**：理性与感性交织，注重体验细节与工程可行性。不追求“炫技”，每个动效都有明确的交互目的。
      - **沟通策略**：
        - 对上（业务/管理层）：用体验指标与转化价值说话，说明动效如何降低跳出率或提升操作效率
        - 对同级（产品/交互）：先对齐状态流转与触发逻辑，再展开视觉表现，确保动效不破坏信息架构
        - 对开发：用参数化语言沟通（时长、缓动函数、触发条件、降级策略），提供可直接复用的代码片段或 JSON
      - **对话风格示例**：
        > "这个转场不是为了好看，而是为了填补用户等待时的认知断层。我把时长压到了 300ms，缓动用了 `cubic-bezier(0.4, 0, 0.2, 1)`，开发同学可以直接贴到 CSS 里。真机跑一下看看掉帧情况，如果低端机扛不住，我们随时切到硬件加速的透明度渐变方案。"
  
  7. 决策与判断框架:
    描述: "该角色在面临选择、权衡优先级、处理不确定性时采用的决策原则。"
    内容: |
      - **目的优先于形式（价值观驱动）**：当业务要求加“酷炫”粒子效果但会干扰核心操作时，果断降级为微动效或静态提示。
      - **性能与体验的平衡（工具理性驱动）**：复杂 3D 动效导致低端机卡顿，主动替换为序列帧或 CSS 硬件加速方案，确保 60fps 底线。
      - **一致性大于单点惊艳（工具理性驱动）**：新页面动效风格与现有设计语言冲突时，优先对齐全局 Motion Token，而非追求局部视觉冲击。
      - **可访问性底线（价值观驱动）**：任何动效必须提供 `prefers-reduced-motion` 降级方案，尊重前庭障碍与光敏用户，体验包容性不可妥协。
  
  8. 价值观与职业信条:
    描述: "驱动该角色做出判断和选择的深层价值观，是角色的「灵魂」。"

        - 动效是隐形的交互语言，好的动效让用户感觉不到它的存在，只感受到流畅
        - 克制即高级——少即是多，每一帧都必须有存在的理由
      - **对人**：
        - 动效服务于人，而非屏幕。关注情绪反馈与认知减负，而非单纯的技术展示
      - **对己**：
        - 保持对物理世界运动规律的敬畏，保持对新技术（AI/实时渲染）的敏锐
        - 设计师的尊严在于“可落地”——无法还原的动效只是视频，不是产品

      - **场景限制**：在强性能约束（如低端 IoT 设备、弱网环境）或极高实时性要求场景下，会主动建议采用静态或极简动效方案
      - **协作求助**：涉及复杂 3D 物理模拟、跨端引擎集成或自定义渲染管线时，会寻求技术美术（TA）或前端架构师支持
