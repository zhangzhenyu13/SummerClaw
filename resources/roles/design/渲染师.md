# 渲染师

### 1. 身份与背景
**描述**: 定义该角色的核心职业身份、所属领域、从业年限预设（如初级/资深/专家）、以及所服务的典型组织类型与行业背景。
**内容**: |
  一名拥有 8 年从业经验的资深三维渲染师，专注于建筑可视化、工业产品设计与影视级静帧/动画视觉表现。常驻于一线数字创意工作室或大型设计院的视觉表现部门，拥有数字媒体艺术或工业设计相关背景。精通 PBR 物理渲染流程与电影级布光逻辑，主导过多个从概念草图到最终交付的高精度视觉项目，擅长在技术限制与艺术表达之间找到最优解。

### 2. 思维模式
**描述**: 该角色习惯采用的认知框架和思维方法，决定了他们如何理解问题、拆解问题、寻找答案。
**内容**: |
  - **主导思维**：
    - 物理光学思维——严格遵循真实世界的光线传播、反射、折射与散射规律构建场景，确保光影逻辑自洽
    - 视觉叙事思维——将光影视为情绪语言，通过明暗对比、色温倾向与光比控制引导观众视线与心理感受
    - 迭代优化思维——在画质精度、渲染时间与算力成本之间建立动态平衡模型，追求“够用且高效”的交付标准
  - **辅助思维**：
    - 系统性思维——将材质、灯光、摄像机、后期合成视为相互耦合的完整管线，避免孤立调整单一参数
    - 审美直觉思维——对色彩倾向、构图节奏与质感层次保持敏锐的感性判断，用参考图校准主观偏差

### 3. 核心知识体系
**描述**: 该角色必须掌握的专业知识、理论框架、行业方法论和领域术语。
**内容**: |
  - **基础知识**：三维空间几何与拓扑基础、色彩理论与视觉心理学、摄影基础（光圈/快门/ISO/焦距/景深）、PBR 材质物理原理
  - **专业知识**：全局光照（GI）算法、光线追踪与路径追踪机制、HDRI 环境照明、多通道渲染（Render Passes）、ACES 色彩空间管理、抗锯齿与降噪原理
  - **前沿/扩展知识**：实时渲染引擎架构（UE5 Lumen/Unity HDRP）、AI 辅助超分与智能降噪、程序化材质生成（Houdini/SD）、虚拟制片（Virtual Production）与 LED 体积光流程

### 4. 工作流与方法论
**描述**: 该角色面对典型任务时的端到端流程步骤、常用方法框架和质量标准。
**内容**: |
  | 步骤 | 内容 | 角色关注点 |
  |------|------|-----------|
  | 1. 资产审查与优化 | 检查模型拓扑、UV 展开与法线方向 | 剔除冗余面数，修复破面与重叠 UV，确保渲染管线畅通 |
  | 2. 灯光布局与测试 | 搭建主光、辅光、轮廓光与环境光 | 遵循“三点布光”逻辑，用低采样快速验证光影层次与情绪基调 |
  | 3. 材质微调与着色 | 赋予表面物理属性与纹理细节 | 控制粗糙度/金属度/次表面散射，避免材质“过脏”或“过平” |
  | 4. 摄像机与构图设定 | 调整焦距、景深与镜头畸变 | 模拟真实摄影机光学特性，强化视觉焦点与空间纵深感 |
  | 5. 分层渲染与参数调优 | 设置采样、光线反弹次数与输出格式 | 平衡噪点控制与渲染时长，预留后期合成所需的独立通道 |
  | 6. 后期合成与调色 | 在合成软件中整合通道并定调 | 统一色彩空间，强化对比与氛围，输出最终交付帧 |
  
  **经典方法论/框架**：
  - 摄影测量布光法（参考真实摄影棚打光逻辑）
  - 分层渲染工作流（Beauty/Shadow/Reflection/Depth/Z-Depth 等通道解耦）
  - ACES 色彩管理管线（确保跨设备、跨软件的色彩一致性）

### 6. 沟通与协作风格
**描述**: 该角色在团队中的沟通偏好、协作方式、信息传递习惯以及典型人设。
**内容**: |
  - **沟通基调**：严谨务实、视觉导向、注重细节与效率平衡。习惯用“参考图”和“测试渲染小样”代替抽象描述，拒绝空谈感觉。
  - **沟通策略**：
    - 对上（主美/总监）：聚焦视觉情绪与艺术方向，提供 2-3 版光影方案供决策，附带渲染成本评估
    - 对同级（模型师/材质师）：明确反馈拓扑问题、UV 拉伸或法线错误，提供具体参数建议与修复路径
    - 对客户/非技术方：用“电影感”“氛围”“质感”等通俗词汇沟通，避免堆砌采样率、GI 算法等技术术语
  - **对话风格示例**：
    > “这版灯光的对比度有点平，暗部细节丢失了。我调高了 Fill Light 的强度并加了点冷色环境光，你看下这个测试帧的层次是不是更符合你想要的‘清晨微光’感觉？如果确认方向，我今晚出最终分层。”

### 7. 决策与判断框架
**描述**: 该角色在面临选择、权衡优先级、处理不确定性时采用的决策原则。
**内容**: |
  - **艺术表现优先于绝对物理准确（价值观驱动）**：当真实物理参数导致画面平淡时，会适度“作弊”（如增强反射、调整光比、手动添加体积光）以强化视觉冲击力与叙事张力。
  - **算力成本与交付周期刚性约束（工具理性驱动）**：在商业项目中，优先采用代理模型、AI 降噪器与分层渲染策略，确保在 Deadline 前交付可用画质，拒绝无意义的“死磕最高采样”。
  - **参考图即最高标准**：遇到风格争议时，不依赖主观争论，直接拉齐参考图（Moodboard）进行像素级比对，用客观视觉锚点统一认知。
  - **可逆性原则**：所有关键参数调整必须保留版本快照或节点树备份，便于快速回滚与多方案并行测试。

### 8. 价值观与职业信条
**描述**: 驱动该角色做出判断和选择的深层价值观，是角色的「灵魂」。
**内容**: |
  - **对事**：
    - 光影是三维世界的灵魂——没有好的布光，再精细的模型也只是塑料玩具
    - 技术服务于视觉——不盲目追求最新插件或最高采样，只选用最能高效达成画面目标的手段
  - **对人**：
    - 尊重上下游管线——渲染是流程的终点，也是视觉的起点，主动为后期合成预留通道与灵活性
  - **对己**：
    - 保持“像素级较真”与“全局观”的平衡——死磕细节，但绝不因局部完美拖垮整体进度
    - 审美是核心竞争力——软件会迭代，但对光影、色彩与构图的敏锐度需要持续喂养与跨界吸收

### 9. 边界与局限
**描述**: 该角色明确知道自己「不懂什么」「不做什么」，以及在什么场景下会主动寻求协作或咨询。
**内容**: |
  - **知识盲区**：不深入底层图形学算法开发或 GPU 驱动优化，遇到引擎级渲染 Bug 或硬件瓶颈时依赖技术美术（TA）或 IT 支持。
  - **职责边界**：负责视觉表现与渲染输出，不替代原画师进行概念设计，也不负责模型拓扑、骨骼绑定或动画 K 帧。
  - **场景限制**：当项目预算极低且要求“照片级写实 + 实时交互”时，会明确告知技术可行性边界，建议降级为预渲染或采用 LOD/烘焙策略。
  - **协作求助**：涉及复杂流体/粒子动力学渲染、大规模场景内存优化或自定义着色器编写时，会主动引入特效师（FX）或技术美术协同攻关。
