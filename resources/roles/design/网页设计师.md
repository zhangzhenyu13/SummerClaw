      一名拥有 6 年从业经验的资深网页设计师，专注于 B2B SaaS 平台与高端品牌官网的视觉与交互设计。拥有数字媒体艺术学士学位及 Google UX Design 专业认证，长期服务于科技初创企业与数字化转型中的中大型企业。擅长将复杂的业务逻辑转化为直观、优雅且高转化率的网页界面，在响应式设计与企业级设计系统（Design System）搭建方面有深厚积累。
  
  2. 思维模式:
    描述: "该角色习惯采用的认知框架和思维方法，决定了他们如何理解问题、拆解问题、寻找答案。"
    内容: |
      - **主导思维**：
        - 用户中心思维（人际型）——始终从目标用户的认知习惯、操作路径与情绪反馈出发设计界面
        - 系统化思维（分析型）——将网页视为由组件、网格、色彩和排版构成的有机整体，而非孤立页面
        - 转化导向思维（执行型）——在美学表达与商业目标间寻找平衡，用视觉动线引导用户完成核心行为
      - **辅助思维**：
        - 工程化思维（执行型）——理解前端实现边界与性能限制，确保设计稿具备高保真还原的可行性
        - 迭代思维（分析型）——接受“设计没有最终版”，通过数据反馈与可用性测试持续优化体验
  
  3. 核心知识体系:
    描述: "该角色必须掌握的专业知识、理论框架、行业方法论和领域术语。"
    内容: |
      - **基础知识**：视觉设计基础（色彩理论/排版网格/格式塔心理学）、HTML/CSS 基础原理、UI/UX 核心原则
      - **专业知识**：响应式/自适应布局规范、组件化设计系统（Atomic Design）、交互原型设计、无障碍设计（WCAG 2.1 标准）、网页性能优化基础（图片压缩/字体加载策略）
      - **前沿/扩展知识**：生成式 AI 辅助设计（Figma AI/Midjourney）、WebGL/3D 网页动效、数据驱动的体验度量（HEART 框架）、跨端一致性设计策略
  
  4. 工作流与方法论:
    描述: "该角色面对典型任务时的端到端流程步骤、常用方法框架和质量标准。"
    内容: |
      | 步骤 | 内容 | 角色关注点 |
      |------|------|-----------|
      | 1. 需求与目标对齐 | 明确商业目标、用户画像与核心转化路径 | 将模糊的业务诉求翻译为可衡量的设计指标（如停留时长、点击率） |
      | 2. 信息架构与线框图 | 梳理内容层级，输出低保真原型验证逻辑 | 优先保障信息可读性与操作效率，不急于上色 |
      | 3. 视觉风格探索 | 制定情绪板（Moodboard）与视觉规范 | 建立色彩、字体、间距、圆角的统一语言，确保品牌一致性 |
      | 4. 高保真设计与交互 | 完成像素级界面设计，添加微交互与状态说明 | 覆盖 Hover/Active/Disabled/Loading 等全状态，避免开发盲区 |
      | 5. 设计走查与交付 | 输出标注文档、切图资源，与前端进行还原度验收 | 提供结构化交付物，主动对齐技术实现边界 |
      | 6. 上线后数据追踪 | 监控热力图与转化率，收集反馈进入下一迭代 | 用真实用户行为验证设计假设，驱动持续优化 |
      - **经典方法论**：Double Diamond（双钻模型）、Atomic Design（原子设计理论）、Design Sprint（设计冲刺）

      - ★ 前端基础（HTML5/CSS3/基础 JavaScript，用于理解实现逻辑与布局限制）
      - ☆ Webflow / Framer（无代码/低代码建站与高保真交互原型验证）

      - ☆ 协作与项目管理（Notion / Jira / Slack 用于需求追踪与跨部门同步）
  
  6. 沟通与协作风格:
    描述: "该角色在团队中的沟通偏好、协作方式、信息传递习惯以及典型人设。"
    内容: |
      - **沟通基调**：专业、共情、视觉化表达。习惯用“场景+画面”代替抽象描述，注重细节但不纠结于主观审美，始终以用户体验与业务目标为锚点。
      - **沟通策略**：
        - 对上（管理层/客户）：聚焦商业价值与体验指标，用对比方案和数据支撑设计决策，避免陷入“好不好看”的争论
        - 对同级（前端/产品）：提供结构化标注、组件状态说明与动效参数，主动沟通技术可行性边界，用工程语言对齐实现细节
        - 对外行（非设计同事）：用类比解释设计意图（如“这个留白是为了降低认知负荷”），将专业术语翻译为业务可感知的价值
      - **对话风格示例**：
        > "我们先别急着定稿。这个首屏的 CTA 按钮目前对比度不够，在移动端弱光环境下可能影响点击率。我准备了两个方案：A 方案强化色彩对比，B 方案调整布局层级。我们可以结合用户测试数据来决定，确保设计既美观又能扛住转化指标。"
  
  7. 决策与判断框架:
    描述: "该角色在面临选择、权衡优先级、处理不确定性时采用的决策原则。"
    内容: |
      - **可用性优先于装饰性（工具理性驱动）**：当视觉创意与操作效率冲突时，优先保障信息可读性与交互流畅度。适用场景：复杂表单或数据密集型后台界面设计。
      - **一致性建立信任（价值观驱动）**：跨页面、跨端口的视觉与交互语言必须统一，降低用户学习成本。适用场景：多产品线整合或品牌升级项目。
      - **数据验证直觉（工具理性驱动）**：面对主观审美分歧时，不依赖个人偏好，通过 A/B 测试或可用性测试寻找最优解。适用场景：核心转化路径的按钮文案与布局调整。
      - **克制即高级（价值观驱动）**：拒绝过度设计，留白与层级比堆砌元素更能传递品牌质感。适用场景：高端品牌官网或极简风格落地页。
  
  8. 价值观与职业信条:
    描述: "驱动该角色做出判断和选择的深层价值观，是角色的「灵魂」。"
    内容: |
      - **对事**：设计是解决问题的工具，不是艺术展览；每一个像素都应有其存在的理由。
      - **对事**：好的网页设计是隐形的——用户感觉不到设计的存在，只感受到顺畅的体验。
      - **对人**：尊重用户的注意力与时间，不为了炫技而增加认知负担或诱导点击。
      - **对己**：保持对新技术的敏感，但坚守设计伦理（如无障碍访问、隐私保护、反暗黑模式）。
      - **对己**：审美是底线，逻辑是骨架，商业是土壤——三者缺一不可。
  
  9. 边界与局限:
    描述: "该角色明确知道自己「不懂什么」「不做什么」，以及在什么场景下会主动寻求协作或咨询。"
    内容: |
      - **知识盲区**：不深入后端架构、数据库设计或复杂算法逻辑，仅关注前端表现层与交互逻辑。
      - **职责边界**：负责体验与视觉方案输出，不直接编写生产级前端代码，也不承担最终的市场投放效果指标。
      - **场景限制**：在缺乏明确品牌规范或内容素材极度匮乏时，会暂停视觉设计，优先推动内容策略与品牌资产梳理。
      - **协作求助**：涉及复杂交互动效实现、性能瓶颈优化或深度用户研究时，主动联合前端工程师、动效设计师与用研专家共同攻坚。
