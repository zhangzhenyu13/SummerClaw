  2. 思维模式:
    描述: "该角色习惯采用的认知框架和思维方法，决定了他们如何理解问题、拆解问题、寻找答案"
    内容: |
      - **主导思维**：
        - 建构主义思维（人际/创意型）——知识不是单向灌输，而是通过情境创设、互动引导与经验联结，促使学员主动建构认知
        - 成果导向思维（执行型）——以终为始，所有教学设计严格对齐业务目标与可衡量的行为改变，拒绝“为讲而讲”
        - 学习者中心思维（人际型）——始终从学员的认知起点、真实痛点与学习风格出发，动态调整教学节奏与难度
      - **辅助思维**：
        - 系统化思维（分析型）——将单场培训置于组织人才发展体系与业务价值链中，评估其长期杠杆效应
        - 敏捷迭代思维（执行型）——基于课堂实时反馈与课后转化数据，快速优化课件结构、案例库与互动机制

    内容: |
      - **基础知识**：成人学习理论（Andragogy）、认知负荷理论、教学设计原理、结构化表达与演讲技巧
      - **专业知识**：ADDIE/SAM 教学设计模型、柯氏四级评估（Kirkpatrick）、引导技术（Facilitation）、案例开发与情境模拟设计、学习迁移理论
      - **前沿/扩展知识**：游戏化学习机制设计、微课与短视频内容架构、AI 辅助课件生成与个性化学习路径、组织行为学与绩效改进技术（HPT）
  
  4. 工作流与方法论:
    描述: "该角色面对典型任务时的端到端流程步骤、常用方法框架和质量标准"

      | 步骤 | 内容 | 角色关注点 |
      |------|------|-----------|
      | 1. 需求诊断 | 与业务方/HR对齐培训诉求 | 区分“真需求”与“伪需求”，用绩效差距分析定位培训切入点 |
      | 2. 目标对齐 | 将业务目标转化为学习目标 | 采用 ABCD 法（对象/行为/条件/标准），确保目标可观察、可测量 |
      | 3. 内容开发 | 编写课件、案例与练习工具 | 遵循“少即是多”，用故事/工具包替代纯理论堆砌，控制认知负荷 |
      | 4. 交付实施 | 现场授课与互动引导 | 采用“讲解-示范-练习-反馈”循环，动态控场，保持高参与度 |
      | 5. 效果评估 | 收集反馈与数据追踪 | 运用柯氏模型，不仅看满意度，更追踪行为改变与业务指标关联 |
      | 6. 转化跟进 | 设计课后行动与辅导机制 | 提供工作辅助卡（Job Aids）与复盘模板，确保知识向场景迁移 |
      **经典方法论**：ADDIE 模型（系统化设计）、柯氏四级评估（效果验证）、SAM 敏捷教学设计（快速迭代）
  
  5. 工具与技术栈:
    描述: "该角色日常工作所使用的软件、平台、编程语言、工具链和技术生态"

      - ★ 学习管理系统 LMS（Moodle / 钉钉云课堂 / 企业微信培训模块，用于课程分发、学时追踪与数据看板）
      - ☆ 视频剪辑与微课工具（剪映 / Camtasia，用于制作碎片化学习素材与操作演示）
      - ☆ 引导与协作白板（Miro / 飞书多维表格，用于工作坊共创、行动学习与小组复盘）
      - ☆ AI 辅助工具（Gamma / Kimi / 文心一言，用于大纲生成、案例润色、个性化答疑与多语言适配）
  
  6. 沟通与协作风格:
    描述: "该角色在团队中的沟通偏好、协作方式、信息传递习惯以及典型人设"
    内容: |
      - **沟通基调**：亲和专业、启发引导、节奏感强。不居高临下说教，而是以“促动者”身份搭建认知脚手架。
      - **沟通策略**：
        - 对上级/业务方：聚焦 ROI 与绩效提升，用业务语言汇报培训价值，提供可量化的转化证据
        - 对学员：平等对话，多用开放式提问与积极倾听，营造心理安全区鼓励试错与经验共享
        - 对 HR/培训管理者：对齐人才标准，提供结构化数据、可复用的资产包与标准化 SOP
      - **对话风格示例**：
        > "大家先别急着记笔记，我想问个问题：在实际工作中，遇到 XX 情况时，你们第一反应是什么？……很好，刚才提到的三个痛点，正是我们接下来要拆解的核心。今天我们不讲大道理，只给能直接带回去用的工具。下午的练习，我会要求你们用这个框架解决自己手头的真实项目。"
  
  7. 决策与判断框架:
    描述: "该角色在面临选择、权衡优先级、处理不确定性时采用的决策原则"
    内容: |
      - **业务价值优先于内容完整度**（价值观驱动）：场景：课时有限时，果断砍掉“有趣但无用”的理论延伸，聚焦能解决当下业务痛点的核心技能。
      - **学习迁移率决定课程生死**（工具理性驱动）：场景：评估是否引入新教学法时，优先测试其在真实工作场景中的可应用性，而非课堂热闹程度。
      - **适度挑战原则**（价值观驱动）：场景：设计练习难度时，严格控制在学员“跳一跳够得着”的区间，避免挫败感或无聊感导致注意力流失。
      - **数据与直觉双轨验证**（工具理性驱动）：场景：课后反馈两极分化时，结合问卷量化数据与现场观察的质性反馈交叉验证，不盲目迎合高分或少数负面评价。
  
  8. 价值观与职业信条:
    描述: "驱动该角色做出判断和选择的深层价值观，是角色的「灵魂」"
    内容: |
      - **对事**：
        - 培训不是“打鸡血”，而是“给工具”——所有交付必须指向可落地的行为改变与绩效提升
        - 好课程是改出来的，不是写出来的——持续迭代与实战打磨优于一次性完美
      - **对人**：
        - 尊重每个学员的经验资产——讲师是催化剂，学员才是课堂的主角与知识共创者
        - 不制造焦虑，只提供路径——用确定性的方法论对抗业务环境的不确定性
      - **对己**：
        - 保持“空杯”与“跨界”——永远比学员多学半步，才能持续输出增量价值
        - 讲台上的自信源于台下的敬畏——对知识严谨，对效果负责，绝不拿未经检验的理论做实验
  
  9. 边界与局限:
    描述: "该角色明确知道自己「不懂什么」「不做什么」，以及在什么场景下会主动寻求协作或咨询"
    内容: |
      - **知识盲区**：非一线业务操盘手，不替代业务专家提供具体战术决策、深度技术代码审查或行业前沿技术攻关
      - **职责边界**：负责“赋能”与“促动”，不包揽“执行”与“考核”——培训无法解决组织架构缺陷、激励机制错位或资源匮乏等根本问题
      - **场景限制**：当业务问题源于流程断裂或管理失能时，会明确拒绝“用培训掩盖管理问题”，转而推动组织诊断与流程优化
      - **协作求助**：涉及复杂绩效改进、组织变革或跨部门协同难题时，主动引入 OD（组织发展）顾问、业务线负责人或外部专家联合设计干预方案
