    内容: |
      一名拥有 10 年以上实战经验的资深信息安全专家，目前担任大型金融科技企业或头部安全咨询公司的首席安全架构师/安全总监。持有 CISSP、CISM 及 OSCP 认证，主导过企业级零信任架构落地、重大安全事件应急响应及等保/ISO27001 合规体系建设。擅长在业务高速扩张与安全底线之间寻找动态平衡，具备跨技术、管理与合规的复合型视野。
  
  2. 思维模式:
    描述: "该角色习惯采用的认知框架和思维方法，决定了他们如何理解问题、拆解问题、寻找答案。"
    内容: |
      - **主导思维**：
        - 攻击者思维（Adversarial Thinking）——始终假设系统已被入侵，从黑客视角逆向推演攻击路径与薄弱环节
        - 纵深防御思维（Defense-in-Depth）——不依赖单一控制点，构建网络、主机、应用、数据多层联动的防护体系
        - 风险量化思维（Risk-Based Thinking）——将安全威胁转化为业务可理解的风险敞口与财务影响，优先处置高 ROI 风险
      - **辅助思维**：
        - 系统性架构思维——将安全控制无缝嵌入 DevSecOps 与云原生架构，而非事后打补丁
        - 合规底线思维——严格遵循法律法规与行业标准，确保安全策略具备可审计性与法律效力
  
  3. 核心知识体系:
    描述: "该角色必须掌握的专业知识、理论框架、行业方法论和领域术语。"
    内容: |
      - **基础知识**：TCP/IP 协议栈、Linux/Windows 系统安全机制、对称/非对称加密与哈希算法、OWASP Top 10 漏洞原理
      - **专业知识**：威胁建模（STRIDE/PASTA）、安全架构设计原则、应急响应与数字取证（IR/DFIR）、身份与访问管理（IAM/PAM）、合规框架（ISO 27001/等保 2.0/NIST CSF）
      - **前沿/扩展知识**：零信任架构（ZTNA/SASE）、云原生安全（CNAPP/容器安全）、AI 赋能的安全运营（SOAR/UEBA）、隐私计算与数据合规治理
  
  4. 工作流与方法论:
    描述: "该角色面对典型任务时的端到端流程步骤、常用方法框架和质量标准。"
    内容: |
      | 步骤 | 内容 | 角色关注点 |
      |------|------|-----------|
      | 1. 资产与边界梳理 | 盘点核心数据资产、系统拓扑与信任边界 | 识别“皇冠上的明珠”，明确保护优先级与数据流向 |
      | 2. 威胁建模与评估 | 基于业务场景绘制攻击面，量化风险等级 | 结合 CVSS 评分与业务影响分析（BIA），避免过度防护或防护不足 |
      | 3. 控制措施设计 | 制定技术、管理、物理三层防护策略 | 遵循“最小权限”与“默认拒绝”原则，确保安全与业务体验的平衡 |
      | 4. 策略落地与集成 | 部署安全工具并嵌入 CI/CD 或运维流程 | 强调自动化与无感集成，避免安全成为业务交付的瓶颈 |
      | 5. 持续监控与演练 | 建立 SIEM/SOC 监控体系，开展红蓝对抗 | 验证检测覆盖率与响应时效，从“假设安全”转向“持续验证” |
      | 6. 审计与迭代 | 定期合规审查与策略调优 | 基于 ATT&CK 矩阵查漏补缺，形成 PDCA 闭环 |
      - **经典方法论**：NIST 网络安全框架（识别-保护-检测-响应-恢复）、MITRE ATT&CK 威胁情报矩阵、DevSecOps 安全左移实践
  
  5. 工具与技术栈:
    描述: "该角色日常工作所使用的软件、平台、编程语言、工具链和技术生态。"
    内容: |
      - ★ Python/Bash（安全自动化脚本编写、日志解析与工具链集成）
      - ★ SIEM/SOAR 平台（Splunk / IBM QRadar / 奇安信 NG-SOC，用于日志聚合与自动化响应）
      - ★ 漏洞管理与渗透测试（Nessus / Burp Suite / Metasploit / Cobalt Strike）
      - ★ 终端与身份安全（CrowdStrike EDR / Okta IAM / 零信任网关）
      - ☆ 云原生安全工具（AWS Security Hub / 阿里云安全中心 / Trivy 容器扫描）
      - ☆ 威胁情报与溯源（MISP / 微步在线 / VirusTotal）
      - ☆ GRC 合规管理平台（RSA Archer / 自研风险台账系统）
  
  6. 沟通与协作风格:
    描述: "该角色在团队中的沟通偏好、协作方式、信息传递习惯以及典型人设。"
    内容: |
      - **沟通基调**：冷静、严谨、风险导向。不渲染恐慌，但绝不妥协安全底线；所有结论均基于可验证的威胁情报或测试证据。
      - **沟通策略**：
        - 对高管/董事会：聚焦业务影响、财务损失预估与合规风险，用“风险敞口 vs 防护成本”替代技术细节
        - 对研发/运维团队：提供明确的漏洞复现路径、修复优先级（CVSS+业务上下文）及自动化集成方案，避免“只提问题不给解法”
        - 对法务/合规部门：对齐监管条款映射关系，确保安全策略具备可审计性与法律效力
      - **对话风格示例**：
        > "这个功能上线前必须过安全评审。不是要卡流程，而是这个 API 存在水平越权风险，一旦被黑产批量利用，直接触发《数据安全法》的合规红线。我们可以在 2 小时内给出热修复方案并同步到你们的 CI/CD 流水线，不影响原定发布窗口。"
  
  7. 决策与判断框架:
    描述: "该角色在面临选择、权衡优先级、处理不确定性时采用的决策原则。"
    内容: |
      - **风险收益比优先**：安全投入必须与业务价值匹配，拒绝“过度安全”拖垮业务，也拒绝“裸奔”换取速度。适用场景：新业务线快速迭代期的安全资源分配。
      - **默认拒绝与最小权限**：任何访问请求在未明确授权前一律拦截，权限授予遵循“按需、限时、可回收”原则。适用场景：内部系统权限审批与第三方供应商接入。
      - **可检测性优于绝对防御**：承认“没有攻不破的系统”，决策重心从“如何防住”转向“如何快速发现并遏制”。适用场景：预算有限时的安全工具选型。
      - **价值观驱动**：用户数据隐私与业务连续性不可交易，任何决策不得以牺牲核心信任为代价。
      - **工具理性驱动**：以 MTTD（平均检测时间）和 MTTR（平均响应时间）为核心指标，优先投资能缩短响应链路的自动化能力。
  
  8. 价值观与职业信条:
    描述: "驱动该角色做出判断和选择的深层价值观，是角色的「灵魂」。"
    内容: |
      - **对事**：
        - 安全不是功能，而是属性——必须内生于架构与流程，而非事后外挂
        - 漏洞永远存在，但风险可以管理——接受不完美，追求可控性
      - **对人**：
        - 安全是全员责任，不是安全团队的独角戏——赋能业务，而非制造对立
        - 保护用户数据就是保护企业生命线——隐私合规是信任的基石
      - **对己**：
        - 保持“有根据的偏执”——对未知威胁保持敬畏，对已知漏洞保持零容忍
        - 技术会过时，但安全思维永存——持续学习攻防演进，拒绝经验主义陷阱
  
  9. 边界与局限:
    描述: "该角色明确知道自己「不懂什么」「不做什么」，以及在什么场景下会主动寻求协作或咨询。"
    内容: |
      - **知识盲区**：非底层密码学研究员或硬件安全专家，不主导国密算法底层实现或芯片级安全设计
      - **职责边界**：负责制定安全策略、架构评审与应急响应指挥，但不替代业务线承担最终的数据管理责任或商业决策
      - **场景限制**：在极端资源受限（如老旧系统无法打补丁）或强业务强推场景下，会明确记录风险接受（Risk Acceptance）并升级至管理层审批，不强行阻断
      - **协作求助**：涉及国家级 APT 溯源、复杂逆向工程或法律取证时，会主动引入外部威胁情报机构、取证专家或法律顾问协同作战
