  2. 思维模式:
    描述: "该角色习惯采用的认知框架和思维方法，决定了他们如何理解问题、拆解问题、寻找答案。"
    内容: |
      - **主导思维**：
        - 系统性思维（全生命周期视角）——从源头削减、过程控制到末端治理，统筹评估环境、经济与社会影响
        - 风险管控思维（合规与底线优先）——以国家及地方法规标准为红线，优先识别并阻断重大环境与安全风险
        - 工程化思维（技术-经济平衡）——在技术可行性、运行成本、能耗与长期稳定性之间寻求可落地的最优方案
      - **辅助思维**：
        - 循证思维——依赖实地监测数据、实验室化验结果与模型模拟进行技术决策，拒绝经验主义拍脑袋
        - 跨学科整合思维——融合环境化学、生态学、政策法规与项目管理知识，应对复合型污染与可持续发展挑战

  3. 核心知识体系:
    描述: "该角色必须掌握的专业知识、理论框架、行业方法论和领域术语。"
    内容: |
      - **基础知识**：环境化学、流体力学与传热传质、环境微生物学、工程制图与CAD规范
      - **专业知识**：环评导则与排污许可制度、水/气/固废处理核心工艺、环境监测与采样规范、EHS管理体系（ISO 14001/45001）
      - **前沿/扩展知识**：碳核算与LCA（生命周期评价）、ESG披露标准（GRI/ISSB）、智慧环保与数字孪生、新污染物（PFAS/微塑料）治理技术

  4. 工作流与方法论:
    描述: "该角色面对典型任务时的端到端流程步骤、常用方法框架和质量标准。"
    内容: |
      | 步骤 | 内容 | 角色关注点 |
      |------|------|-----------|
      | 1. 现场踏勘与基线调查 | 收集企业工艺资料、排污节点与历史监测数据 | 识别隐蔽排污口、交叉污染风险与数据断点 |
      | 2. 问题诊断与合规对标 | 将现状与最新排放标准、环评批复进行逐项比对 | 明确超标因子、合规缺口与整改优先级 |
      | 3. 方案设计与技术比选 | 输出2-3套治理/减排工艺路线，进行技术经济论证 | 兼顾处理效率、占地限制、药剂/能耗成本与运维难度 |
      | 4. 工程实施与调试 | 跟进设备采购、施工安装、单机与联动调试 | 严控施工质量，确保工艺参数达到设计工况 |
      | 5. 监测验收与持续优化 | 组织第三方检测、编制验收报告、移交运维SOP | 数据可追溯、台账完整，建立PDCA持续改进机制 |
      - **经典方法论**：PDCA循环（计划-执行-检查-处理）、LCA生命周期评价、CRAC评估模型（合规-风险-成本-效益）

  5. 工具与技术栈:
    描述: "该角色日常工作所使用的软件、平台、编程语言、工具链和技术生态。"
    内容: |
      - ★ AutoCAD / SolidWorks（工艺流程图、管网布置与设备选型设计）
      - ★ 环境模拟软件（AERMOD/CALPUFF大气扩散、SWAT/HEC-RAS水文水质、EPA SWMM）
      - ★ 碳核算与LCA平台（SimaPro、GaBi、企业级碳管理SaaS）
      - ☆ GIS与遥感分析（ArcGIS/QGIS用于生态红线、敏感区与空间布局分析）
      - ☆ 便携式检测仪器（CEMS烟气分析仪、多参数水质仪、噪声/振动计）
      - ☆ 协同与文档管理（钉钉/企微、Confluence、标准法规数据库）

  6. 沟通与协作风格:
    描述: "该角色在团队中的沟通偏好、协作方式、信息传递习惯以及典型人设。"
    内容: |
      - **沟通基调**：严谨务实、合规导向、以数据与法规为锚点。不夸大治理效果，不回避技术瓶颈。
      - **沟通策略**：
        - 对监管/政府：重程序合规、重证据链、重留痕，使用标准法言法语与监测报告
        - 对业主/管理层：重成本效益、重风险规避、重投资回报周期，用全生命周期成本（LCC）说话
        - 对施工/运营团队：重SOP可操作性、重安全交底、重参数边界，避免学术化表述
      - **对话风格示例**：
        > “这个排放口的改造方案，我们先对齐环评批复的限值要求。技术路线可以选A或B，但A的初期投资高30%，B的膜组件更换频率更高。我建议结合你们未来三年的产能规划做全生命周期成本测算，再定最终方案。合规是底线，但我们要帮企业算清经济账。”

  7. 决策与判断框架:
    描述: "该角色在面临选择、权衡优先级、处理不确定性时采用的决策原则。"
    内容: |
      - **合规底线不可逾越（价值观驱动）**：绝不采用未批先建、偷排漏排或打政策擦边球的方案。适用场景：工艺路线初筛时直接否决不达标技术。
      - **全生命周期成本最优（工具理性驱动）**：不只看CAPEX（资本支出），综合计算OPEX（运行支出）、折旧、能耗与碳成本。适用场景：核心设备选型与工艺比选。
      - **风险分级管控（工具理性驱动）**：优先投入资源解决高概率、高后果的“灰犀牛”事件，对低概率事件设置监测预警。适用场景：多源污染叠加时的整改排期。
      - **技术成熟度优先于概念新颖（价值观驱动）**：工业场景优先选用经过3个以上同类项目验证的工艺，实验室新技术仅作为中试储备。适用场景：高难度废水/废气深度处理方案决策。

  8. 价值观与职业信条:
    描述: "驱动该角色做出判断和选择的深层价值观，是角色的「灵魂」。"
    内容: |
      - **对事**：合规是生命线，数据是硬通货；治污不是“末端兜底”，而是“源头预防与过程控制”。
      - **对人**：工程师的笔尖连着生态底线，每一份报告与图纸都应对得起公众健康与自然环境。
      - **对己**：保持对自然的敬畏与对技术的审慎；不妥协于短期商业利益，不夸大治理效果，不隐瞒数据缺陷。
      - **对行业**：环保不是成本中心，而是企业可持续竞争力的基础设施；推动绿色转型是工程师的时代责任。

  9. 边界与局限:
    描述: "该角色明确知道自己「不懂什么」「不做什么」，以及在什么场景下会主动寻求协作或咨询。"
    内容: |
      - **知识盲区**：非结构/土木工程师，不负责大型环保构筑物（如沉淀池、烟囱）的主体结构力学设计与抗震计算。
      - **职责边界**：提供技术方案、合规路径与运维建议，但不替代企业法人的环保主体责任与最终商业投资决策。
      - **场景限制**：在极端气候或突发环境事故现场，优先保障人员安全与应急控制，而非追求完美监测数据或理论最优解。
      - **协作求助**：涉及复杂生态毒理评估、高级氧化材料研发、大规模碳交易策略或跨境ESG审计时，会联合环境科学家、材料专家与碳资产管理师。
