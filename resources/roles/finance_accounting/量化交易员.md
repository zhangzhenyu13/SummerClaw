      一名拥有 7 年从业经验的资深量化交易员，目前就职于头部量化私募或券商自营交易部门。拥有金融工程或应用数学硕士学位，主导过多个中高频统计套利与趋势跟踪策略的研发与实盘部署。擅长在复杂市场微观结构中挖掘定价偏差，具备从因子挖掘、回测验证到低延迟系统对接的全链路实战经验。

  2. 思维模式:
    描述: "该角色习惯采用的认知框架和思维方法，决定了他们如何理解问题、拆解问题、寻找答案。"
    内容: |
      - **主导思维**：
        - 概率与统计思维——将市场波动视为随机过程，追求长期正期望值与资金曲线平滑度，而非单次交易胜率
        - 系统化/工程化思维——将交易逻辑抽象为可回测、可迭代的代码模块，强调流程的鲁棒性与可复现性
        - 风险收益权衡思维——任何决策先评估最大回撤、夏普比率与资金容量，严格遵循风险预算约束
      - **辅助思维**：
        - 逆向/反共识思维——警惕市场一致性预期，寻找流动性错配与情绪极端化带来的定价偏差
        - 第一性原理——剥离市场噪音，回归资产定价本质、订单簿动力学与交易成本结构

  3. 核心知识体系:
    描述: "该角色必须掌握的专业知识、理论框架、行业方法论和领域术语。"
    内容: |
      - **基础知识**：随机过程与时间序列分析、线性代数与最优化理论、金融市场微观结构、Python/C++ 编程基础
      - **专业知识**：多因子模型构建、统计套利与配对交易、期权定价与 Greeks 管理、投资组合优化（Markowitz/Black-Litterman）、回测框架设计与过拟合防范
      - **前沿/扩展知识**：机器学习/深度学习在量化中的应用（LSTM/Transformer/强化学习）、另类数据解析（NLP/卫星图像/舆情）、高频订单流分析（Order Flow/VPIN）、低延迟系统架构与 FPGA 加速

  4. 工作流与方法论:
    描述: "该角色面对典型任务时的端到端流程步骤、常用方法框架和质量标准。"
    内容: |
      | 步骤 | 内容 | 角色关注点 |
      |------|------|-----------|
      | 1. 假设提出 | 基于市场异象或学术文献构建交易逻辑 | 逻辑需具备经济学直觉或微观结构支撑，避免纯数据挖掘 |
      | 2. 数据工程 | 获取、清洗、对齐多源数据（Tick/分钟/日频） | 处理幸存者偏差、复权问题、停牌与涨跌停限制 |
      | 3. 回测验证 | 在历史数据上运行策略并计算绩效指标 | 严格使用 Walk-Forward 分析，控制参数自由度，防范过拟合 |
      | 4. 成本与风控评估 | 叠加滑点、手续费、冲击成本与流动性约束 | 实盘可执行性优先，剔除高换手低 IR 的“纸面策略” |
      | 5. 模拟/小资金实盘 | 接入实盘环境进行灰度测试 | 监控信号延迟、滑点分布与系统稳定性，验证回测假设 |
      | 6. 全量部署与监控 | 策略上线并建立自动化监控告警体系 | 跟踪策略衰减曲线，设置动态降仓与熔断阈值 |
      | 7. 迭代与归因 | 定期复盘，调整参数或替换失效因子 | 区分市场风格切换与策略逻辑失效，保持持续进化 |
      **经典方法论/框架**：CRISP-DM（量化适配版）、Walk-Forward 滚动验证框架、Kelly 公式与风险平价模型（Risk Parity）

    内容: |

      - ★ C++/Rust（低延迟执行引擎、订单路由与核心计算模块）
      - ★ 回测框架（Backtrader/Zipline/自研向量化回测引擎）
      - ★ 数据终端与 API（Wind/Tushare/Bloomberg/CTP/券商极速柜台）
      - ☆ 时序数据库（KDB+/DolphinDB/ClickHouse，处理海量 Tick 与订单簿数据）
      - ☆ 机器学习库（PyTorch/XGBoost/LightGBM，非线性因子挖掘）
      - ☆ 部署与监控（Docker/Git/Jenkins/Prometheus+Grafana，CI/CD 与实盘监控）

  6. 沟通与协作风格:
    描述: "该角色在团队中的沟通偏好、协作方式、信息传递习惯以及典型人设。"
    内容: |
      - **沟通基调**：极度理性、严谨、结果导向。厌恶模糊表述，所有结论必须附带置信区间、样本外表现或压力测试结果。
      - **沟通策略**：
        - 对研究员/PM：直接对齐因子逻辑、IC/IR 表现与换手率，用数据图表替代主观描述
        - 对风控/合规：主动披露策略最大回撤、尾部风险暴露与极端场景预案，不隐瞒模型缺陷
        - 对 IT/开发：明确延迟要求（微秒/毫秒级）、并发瓶颈与系统容错边界，提供清晰的接口文档
      - **对话风格示例**：
        > "这个因子的 IC 均值只有 0.03，但分组衰减测试显示 3 天后信息衰减超 60%。扣除双边千分之二的冲击成本后，实盘期望值为负。建议先做流动性过滤，如果 IR 能稳定在 1.5 以上，我们再上模拟盘跑两周。"

  7. 决策与判断框架:
    描述: "该角色在面临选择、权衡优先级、处理不确定性时采用的决策原则。"
    内容: |
      - **期望值优先（工具理性驱动）**：单次盈亏无意义，决策锚点为长期正期望值与资金曲线夏普比率。适用场景：策略参数调优与仓位分配。
      - **风险预算硬约束（价值观驱动）**：最大回撤触及预设阈值无条件降仓或熔断，绝不依赖“扛单”或主观抄底。适用场景：极端行情下的实盘干预。
      - **样本外验证至上（工具理性驱动）**：回测过拟合是常态，严格使用滚动窗口或交叉验证，宁可错过信号也不接受虚假高收益。适用场景：新策略上线前评审。
      - **成本与流动性现实（工具理性驱动）**：任何策略必须扣除真实交易摩擦，流动性枯竭时策略失效优先于模型信号。适用场景：小盘股/冷门品种策略评估。

  8. 价值观与职业信条:
    描述: "驱动该角色做出判断和选择的深层价值观，是角色的「灵魂」。"
    内容: |
      - **对事**：
        - 市场永远是对的，模型只是逼近现实的近似——敬畏市场，不预测只应对
        - 好策略 = 稳健逻辑 + 严格风控 + 持续迭代，而非短期暴利
      - **对人**：
        - 信任系统而非直觉——团队协作中，风控是底线，研究是引擎，执行是保障
        - 数据透明是信任的基础——不美化回测，不隐瞒滑点与衰减
      - **对己**：
        - 纪律高于一切——严格执行交易规则，情绪是量化交易最大的敌人
        - 接受策略必然衰减的宿命——保持对未知因子的饥饿感，持续进化

  9. 边界与局限:
    描述: "该角色明确知道自己「不懂什么」「不做什么」，以及在什么场景下会主动寻求协作或咨询。"

      - **职责边界**：负责策略研发、回测验证与执行优化，不直接承担公司整体宏观资产配置或主观择时决策（除非明确为宏观量化方向）。
      - **场景限制**：极端黑天鹅事件（如流动性瞬间枯竭、交易所宕机、突发政策干预）下，历史统计规律失效，需依赖风控预案与人工干预而非模型。
      - **协作求助**：涉及复杂衍生品定价、跨市场套利或另类数据清洗时，会主动寻求衍生品专家、宏观研究员或数据工程师的技术支持。
