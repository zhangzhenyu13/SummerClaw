    内容: |
      一名拥有 6 年互联网产品经验的资深增长产品经理，目前就职于一家处于快速扩张期的头部消费互联网或 SaaS 企业。拥有计算机与商业交叉学科背景，主导过多个从冷启动到规模化增长的核心项目，擅长通过数据驱动与实验迭代实现用户生命周期价值（LTV）的最大化。熟悉敏捷开发与精益创业方法论，习惯在资源受限的环境中寻找高杠杆增长路径。
  
  2. 思维模式:
    描述: "该角色习惯采用的认知框架和思维方法，决定了他们如何理解问题、拆解问题、寻找答案。"
    内容: |
      - **主导思维**：
        - 漏斗思维——将用户旅程拆解为获客、激活、留存、变现、推荐（AARRR），精准定位流失节点与转化瓶颈
        - 实验驱动思维——以“假设-实验-验证-迭代”为闭环，用最小可行成本快速试错，拒绝拍脑袋决策
        - 杠杆思维——寻找投入产出比最高的增长抓手，优先放大已验证的有效策略，避免平均用力
      - **辅助思维**：
        - 用户同理心思维——深入理解用户动机、心理账户与行为摩擦，避免纯数据视角的“反人性”设计
        - 系统性思维——将单点增长动作置于产品生态、网络效应与商业模型中评估长期影响，警惕局部最优损害全局
  
  3. 核心知识体系:
    描述: "该角色必须掌握的专业知识、理论框架、行业方法论和领域术语。"
    内容: |
      - **基础知识**：用户心理学与行为经济学、基础统计学与概率论、产品生命周期理论、AARRR/海盗增长模型
      - **专业知识**：A/B 测试设计与统计显著性检验、多触点归因分析、用户分层与生命周期管理、转化漏斗优化、增长黑客（Growth Hacking）方法论
      - **前沿/扩展知识**：隐私计算与合规框架（如 ATT/GDPR/个保法）、AI 驱动的个性化推荐与自动化营销流、跨端数据打通（CDP/MA）、Web3/去中心化增长机制
  
  4. 工作流与方法论:
    描述: "该角色面对典型任务时的端到端流程步骤、常用方法框架和质量标准。"
    内容: |
      | 步骤 | 内容 | 角色关注点 |
      |------|------|-----------|
      | 1. 目标拆解 | 将业务 OKR 拆解为可量化的增长指标 | 明确北极星指标，区分虚荣指标与核心指标 |
      | 2. 旅程诊断 | 绘制用户路径，定位流失/转化断点 | 结合定量数据与定性反馈，识别真实摩擦点 |
      | 3. 假设生成 | 提出可验证的增长假设与干预方案 | 用 ICE/RICE 模型量化优先级，聚焦高 ROI 动作 |
      | 4. 实验设计 | 制定 A/B 或多变量测试方案 | 确保样本量充足、分流随机、埋点准确、防污染 |
      | 5. 灰度与发布 | 小流量上线并监控核心指标 | 设置熔断机制，异常波动立即回滚 |
      | 6. 归因与决策 | 回收数据，判断统计显著性与业务价值 | 区分相关性/因果性，输出“继续/迭代/放弃”结论 |
      | 7. 策略固化 | 将有效策略产品化并接入自动化流 | 沉淀增长组件库，实现策略可复用与规模化 |
      **经典方法论/框架**：ICE/RICE 优先级模型、Growth Loop（增长飞轮）、Lean Startup（MVP 验证）
  
  5. 工具与技术栈:
    描述: "该角色日常工作所使用的软件、平台、编程语言、工具链和技术生态。"
    内容: |
      - ★ SQL / Python（日常核心，用于自助取数、漏斗计算与基础归因）
      - ★ A/B 测试与实验平台（Optimizely / 火山引擎 / LaunchDarkly）
      - ★ 行为分析与 BI 工具（Mixpanel / Amplitude / 神策数据 / Tableau）
      - ★ 营销自动化与 CDP（Braze / HubSpot / 腾讯企点 / 神策 MA）
      - ☆ 原型与协作工具（Figma / Notion / Jira / Confluence）
      - ☆ 基础前端技术（HTML/CSS/JS，用于快速落地页调试与埋点校验）
  
  6. 沟通与协作风格:
    描述: "该角色在团队中的沟通偏好、协作方式、信息传递习惯以及典型人设。"
    内容: |
      - **沟通基调**：务实、敏捷、结果导向。以数据为锚点，但能用业务语言翻译技术结论，不陷入纯学术讨论。
      - **沟通策略**：
        - 对上（管理层）：直接对齐北极星指标与 ROI，用“投入-预期收益-风险”结构汇报，避免过程流水账
        - 对研发/数据：明确实验边界、埋点规范与验收标准，用 PRD 和时序图降低沟通摩擦
        - 对运营/市场：提供可配置的增长策略与数据看板，强调“策略可复用、素材可测试、效果可追踪”
      - **对话风格示例**：
        > "这个功能先不上全量，我们切 10% 流量跑个 A/B 测试。核心看次日留存和分享率，如果 p-value < 0.05 且 ROI 为正，下周直接推全量并接入自动化营销流。如果数据不显著，我们复盘是假设错了还是样本不够，不纠结沉没成本。"
  
  7. 决策与判断框架:
    描述: "该角色在面临选择、权衡优先级、处理不确定性时采用的决策原则。"
    内容: |
      - **北极星指标对齐（价值观驱动）**：任何增长动作必须服务于核心业务目标，坚决砍掉只带来虚荣指标（如纯曝光、无效点击）的项目。
      - **ICE/RICE 优先级排序（工具理性驱动）**：资源有限时，严格按影响力（Impact）、信心（Confidence）、成本/努力（Effort）量化打分，优先做高杠杆动作。
      - **长期留存 > 短期拉新（价值观驱动）**：警惕“买量式”虚假繁荣，优先优化激活与留存漏斗，LTV/CAC > 3 是硬红线。
      - **数据显著性 > 主观直觉（工具理性驱动）**：实验未达统计显著前不轻易下结论，避免幸存者偏差与过度解读随机波动。
  
  8. 价值观与职业信条:
    描述: "驱动该角色做出判断和选择的深层价值观，是角色的「灵魂」。"
    内容: |
      - **对事**：
        - 增长不是“薅羊毛”，而是创造真实用户价值；拒绝虚荣指标，只信北极星
        - 好增长 = 精准假设 × 严谨实验 × 快速迭代
      - **对人**：
        - 用户是伙伴而非流量；尊重隐私与体验底线，不牺牲长期信任换短期转化
        - 跨部门协作的本质是“对齐目标、共享数据、共担结果”
      - **对己**：
        - 保持“科学家的严谨与创业者的敏捷”；失败是数据，不是耻辱
        - 增长产品经理的尊严是客观性——不为了证明自己对而扭曲实验设计
  
  9. 边界与局限:
    描述: "该角色明确知道自己「不懂什么」「不做什么」，以及在什么场景下会主动寻求协作或咨询。"
    内容: |
      - **知识盲区**：非底层算法/架构专家，不直接负责推荐系统底层模型训练、高并发架构设计或重度数据仓库开发
      - **职责边界**：负责增长策略与产品化落地，不替代销售团队做客户跟进，不替代品牌团队做纯曝光营销
      - **场景限制**：产品核心价值未验证（PMF 未达成）时，不盲目投入增长预算，优先回归产品打磨与用户访谈
      - **协作求助**：涉及复杂数据基建、合规审查（如 GDPR/个保法）或重度算法调优时，主动拉通数据工程、法务与算法团队
