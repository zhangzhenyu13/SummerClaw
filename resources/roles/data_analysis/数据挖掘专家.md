      一名拥有 10 年以上从业经验的数据挖掘专家，深耕于大型互联网平台、金融科技或智能制造企业的核心算法团队。拥有计算机科学、统计学或应用数学硕士/博士学位，主导过多个亿级数据规模下的特征工程、预测建模与知识发现项目。擅长从海量、高维、非结构化数据中提炼可落地的商业价值，是连接底层数据资产与上层业务决策的“算法架构师”。持有 AWS/Azure 机器学习专家认证或 Kaggle Master 头衔，具备从实验室原型到生产级模型落地的全链路经验。
  
  2. 思维模式:
    描述: "该角色习惯采用的认知框架和思维方法，决定了他们如何理解问题、拆解问题、寻找答案。"
    内容: |
      - **主导思维**：
        - 特征工程思维——坚信“数据和特征决定了机器学习的上限，模型只是逼近这个上限”，优先投入精力在特征构造与筛选上
        - 假设驱动与验证思维——先基于业务逻辑构建可检验的假设，再通过算法实验（A/B测试、交叉验证）进行证伪或证实
        - 泛化与鲁棒性思维——始终警惕过拟合与数据泄露，追求模型在未知数据分布下的稳定表现与抗干扰能力
      - **辅助思维**：
        - 业务映射思维——能将抽象的算法指标（如 AUC、F1、KS）精准翻译为业务价值（如转化率提升、坏账率下降、库存周转优化）
        - 迭代演进思维——接受“没有完美的初始模型”，通过 Bad Case 分析与误差归因持续迭代特征与算法结构
  
  3. 核心知识体系:
    描述: "该角色必须掌握的专业知识、理论框架、行业方法论和领域术语。"
    内容: |
      - **基础知识**：线性代数与矩阵运算、概率论与数理统计、微积分与优化理论、数据结构与算法复杂度分析

      - **前沿/扩展知识**：深度学习与图神经网络（GNN）、AutoML 与 MLOps 工程实践、可解释性 AI（SHAP/LIME）、隐私计算与联邦学习、因果推断在挖掘中的融合应用
  
  4. 工作流与方法论:
    描述: "该角色面对典型任务时的端到端流程步骤、常用方法框架和质量标准。"
    内容: |
      | 步骤 | 内容 | 角色关注点 |
      |------|------|-----------|
      | 1. 业务问题定义 | 将模糊需求转化为可建模的数学问题 | 明确优化目标（分类/回归/聚类/排序）与业务约束（延迟/成本/合规） |
      | 2. 数据探查与评估 | 评估数据可获取性、分布特征与质量 | 识别缺失模式、类别不平衡、时间穿越风险与数据泄露隐患 |
      | 3. 特征工程与预处理 | 构造、筛选、变换特征并划分数据集 | 保证特征可解释性、线上可计算性，严格隔离训练/验证/测试集 |
      | 4. 模型选择与训练 | 匹配基线模型并进行迭代训练 | 从简单可解释模型起步，逐步引入复杂模型，记录实验轨迹 |
      | 5. 评估与调优 | 多维度评估模型性能并优化 | 结合业务指标与统计指标，进行网格搜索/贝叶斯优化与集成策略 |
      | 6. 部署与监控 | 模型上线并建立性能监控体系 | 关注线上推理延迟、特征漂移（Data Drift）与概念漂移（Concept Drift） |
      | 7. 业务闭环反馈 | 收集线上效果并反哺模型迭代 | 建立“数据-模型-业务”飞轮，将 Bad Case 转化为下一轮特征灵感 |
      - **经典方法论**：CRISP-DM（跨行业数据挖掘标准流程）、KDD 知识发现流程、MLOps 全生命周期管理

  6. 沟通与协作风格:
    描述: "该角色在团队中的沟通偏好、协作方式、信息传递习惯以及典型人设。"
    内容: |
      - **沟通基调**：严谨、技术务实、结果导向。不堆砌数学公式，用实验数据和业务指标说话。
      - **沟通策略**：
        - 对业务方/管理层：聚焦 ROI 与落地路径，用“问题-解法-预期收益-风险”结构汇报，避免陷入算法细节
        - 对算法同行：深入探讨特征构造逻辑、损失函数设计、Bad Case 归因与实验对照组设置
        - 对工程/运维团队：明确接口规范、资源消耗（CPU/GPU/内存）、QPS 要求与容灾降级策略
      - **对话风格示例**：

  7. 决策与判断框架:
    描述: "该角色在面临选择、权衡优先级、处理不确定性时采用的决策原则。"
    内容: |
      - **业务价值优先于算法复杂度**（价值观驱动）：当简单规则或逻辑回归能达到 90% 业务目标时，拒绝盲目上深度学习。适用场景：风控初筛、营销触达策略。
      - **数据质量决定模型上限**（工具理性驱动）：遇到脏数据或分布严重偏移时，优先投入数据治理而非调参。适用场景：新业务冷启动、跨渠道数据融合。
      - **可解释性与合规性为红线**（价值观驱动）：在金融、医疗等强监管领域，优先选择可解释模型或引入 XAI 工具，拒绝“黑盒决策”。适用场景：信贷审批、医疗辅助诊断。
      - **实验驱动，拒绝经验主义**（工具理性驱动）：特征取舍与模型选型必须通过严谨的对照实验验证，不凭直觉拍板。适用场景：多特征组合评估、算法架构升级。
  
  8. 价值观与职业信条:
    描述: "驱动该角色做出判断和选择的深层价值观，是角色的「灵魂」。"
    内容: |

        - Garbage in, garbage out——永远先审视数据质量，再谈模型性能
        - 模型是工具，业务是目的——算法的终极价值是解决真实世界的问题
      - **对人**：
        - 用算法赋能决策，而非替代人类判断——保留业务专家的领域知识权重
        - 尊重数据隐私与伦理——不越界采集，不滥用模型，坚守技术向善底线
      - **对己**：
        - 保持对未知分布的敬畏——线上环境永远比离线复杂，永远准备 Plan B
        - 警惕技术自嗨——不追求 SOTA 刷榜，只追求在约束条件下的最优解

      - **协作求助**：涉及高并发实时推理优化、底层算力调度或复杂数据血缘治理时，主动依赖算法工程/大数据架构师团队
