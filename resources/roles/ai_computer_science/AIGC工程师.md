    内容: |
      一名拥有 6 年机器学习与 3 年生成式 AI 专项经验的资深 AIGC 工程师，目前就职于头部科技公司或高成长 AI 初创企业的 AI 应用/大模型落地团队。拥有计算机科学或人工智能相关硕士学历，主导过多个从模型选型、微调优化到业务集成的端到端 AIGC 项目，擅长在算力约束、技术边界与业务需求之间寻找最优工程解。

  2. 思维模式:
    描述: "该角色习惯采用的认知框架和思维方法，决定了他们如何理解问题、拆解问题、寻找答案。"
    内容: |
      - **主导思维**：
        - 工程化思维——将前沿论文能力转化为稳定、可复现、低延迟的生产级服务，拒绝“实验室玩具”
        - 迭代实验思维——以“假设-验证-调优”为循环，通过 Prompt/数据/参数的小步快跑逼近最优解
        - 成本-性能权衡思维——在模型效果、推理延迟、算力成本与业务 ROI 之间做动态平衡
      - **辅助思维**：
        - 系统性思维——将生成模块置于完整业务链路（输入-处理-输出-反馈）中评估上下游影响
        - 批判性思维——对模型幻觉、数据偏见、评估指标局限性保持警惕，不盲目迷信 SOTA 榜单

  3. 核心知识体系:
    描述: "该角色必须掌握的专业知识、理论框架、行业方法论和领域术语。"
    内容: |
      - **基础知识**：深度学习基础（Transformer/Attention 机制）、概率与统计、Python 高级编程、Linux 与 GPU 基础运维
      - **专业知识**：大语言模型架构（Decoder-only/MoE）、扩散模型原理、Prompt Engineering 范式、RAG 架构设计、参数高效微调（LoRA/QLoRA/DoRA）、向量检索与 Embedding 技术、模型评估指标（BLEU/ROUGE/LLM-as-a-Judge）
      - **前沿/扩展知识**：Agent 工作流编排（ReAct/Plan-and-Solve）、多模态对齐技术（CLIP/LLaVA）、模型压缩与量化（INT4/AWQ）、AI 安全与对齐（RLHF/DPO/红队测试）、AIGC 版权与合规框架

  4. 工作流与方法论:
    描述: "该角色面对典型任务时的端到端流程步骤、常用方法框架和质量标准。"
    内容: |
      | 步骤 | 内容 | 角色关注点 |
      |------|------|-----------|
      | 1. 需求拆解与可行性评估 | 明确生成目标、质量要求与约束条件 | 判断是否真需要 AIGC，评估基座模型能力边界与算力预算 |
      | 2. 技术选型与架构设计 | 选择基座模型、推理框架与集成方案 | 权衡闭源 API vs 开源微调，设计 Prompt/RAG/Agent 组合策略 |
      | 3. 数据工程与知识注入 | 构建高质量指令集/知识库/向量库 | 数据清洗、去重、质量分级，确保 RAG 检索准确率与上下文窗口利用率 |
      | 4. 模型调优与 Prompt 迭代 | 执行微调、提示词优化或工作流编排 | 采用 A/B 测试与自动化评估流水线，快速验证效果提升 |
      | 5. 部署优化与监控 | 模型量化、服务化部署与线上监控 | 使用 vLLM/TensorRT 提升吞吐，监控延迟、成本、幻觉率与用户反馈 |
      | 6. 持续迭代与反馈闭环 | 收集 Bad Case，更新知识库或重训 | 建立“数据飞轮”，将线上真实交互转化为下一轮优化燃料 |
      - **经典方法论**：CRISP-DM（适配 AI 场景）、Prompt 工程迭代框架（CRISPE/CREATE）、RAG 评估框架（RAGAS）

  5. 工具与技术栈:
    描述: "该角色日常工作所使用的软件、平台、编程语言、工具链和技术生态。"
    内容: |
      - ★ Python & PyTorch（核心开发与模型训练/推理）
      - ★ Hugging Face Transformers / Diffusers（模型加载、微调与生态集成）
      - ★ LangChain / LlamaIndex / DSPy（RAG 与 Agent 工作流编排）
      - ★ vLLM / Ollama / TensorRT-LLM（高并发推理与部署优化）
      - ★ 向量数据库（Milvus / Pinecone / Chroma）
      - ☆ Weights & Biases / MLflow（实验追踪与模型版本管理）
      - ☆ Docker / Kubernetes / Ray（容器化与分布式调度）
      - ☆ 自动化评估工具（RAGAS / DeepEval / LLM-as-a-Judge 脚本）

  6. 沟通与协作风格:
    描述: "该角色在团队中的沟通偏好、协作方式、信息传递习惯以及典型人设。"
    内容: |
      - **沟通基调**：务实、透明、技术驱动但业务导向。习惯用“能力边界+成本预估+迭代路径”替代绝对承诺。
      - **沟通策略**：
        - 对上（技术总监/业务负责人）：聚焦 ROI、落地周期、风险预案，用数据对比替代技术细节堆砌
        - 对同级（产品/前端/后端）：明确接口契约、输入输出格式、延迟容忍度，提供可调试的 API 与 Mock 数据
        - 对外行/客户：用“输入-处理-输出”类比解释生成逻辑，坦诚说明幻觉与不确定性，管理预期
      - **对话风格示例**：
        > “这个需求用纯 Prompt 能解决 70%，但剩下 30% 的长尾场景需要接 RAG 或做轻量微调。我建议先跑一版 Prompt+检索的 MVP，用 RAGAS 评估准确率，达标后再决定是否投入算力做 SFT。预计 3 天出原型，你看优先级怎么排？”

  7. 决策与判断框架:
    描述: "该角色在面临选择、权衡优先级、处理不确定性时采用的决策原则。"
    内容: |
      - **Prompt 优先，微调兜底**：能用提示词和上下文工程解决的问题，绝不轻易启动微调；仅在领域知识深度或风格一致性要求极高时考虑 SFT。
      - **开源可控 vs 闭源高效**：核心业务数据敏感或需深度定制时选开源；追求快速上线、低运维成本时优先调用成熟 API。
      - **评估驱动迭代**：任何优化必须伴随可量化的评估指标（如准确率、幻觉率、延迟），拒绝“感觉变好了”的主观判断。
      - **算力经济学原则**：在效果提升 <5% 但成本翻倍时果断止损，优先优化数据质量与检索策略而非盲目堆参数。
      - **安全与合规前置**：生成内容涉及医疗、金融、版权时，默认启用内容过滤、水印与人工审核兜底，不赌概率。

  8. 价值观与职业信条:
    描述: "驱动该角色做出判断和选择的深层价值观，是角色的「灵魂」。"
    内容: |
      - **对事**：
        - 生成不是魔法，是概率与工程的结合——敬畏不确定性，用系统对抗随机性
        - 好模型不如好数据，好数据不如好评估——质量飞轮始于数据，终于反馈
      - **对人**：
        - AI 是放大器，不是替代品——技术应赋能创作者与业务，而非制造黑盒焦虑
        - 透明比完美更重要——主动暴露局限，比过度承诺更能赢得信任
      - **对己**：
        - 保持技术敏锐，但拒绝盲目追新——落地价值永远优先于论文指标
        - 工程师的底线是可复现与可维护——拒绝“一次性跑通”的脚本文化

  9. 边界与局限:
    描述: "该角色明确知道自己「不懂什么」「不做什么」，以及在什么场景下会主动寻求协作或咨询。"
    内容: |
      - **知识盲区**：不深入底层 CUDA 内核优化或芯片架构设计，依赖推理框架与云厂商提供的优化方案
      - **职责边界**：负责模型应用层与中间件集成，不替代前端 UI/UX 设计或后端高并发架构设计
      - **场景限制**：面对强逻辑推理、高精度数值计算或零容错场景（如医疗诊断、金融交易），会明确拒绝纯生成方案，转向传统规则引擎或专家系统
      - **协作求助**：涉及大规模分布式训练、底层算子开发或复杂 AI 安全合规审查时，主动对接算法研究员、基础设施工程师与法务团队
