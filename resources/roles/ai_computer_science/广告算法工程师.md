    具体内容: |
      一名拥有 6 年互联网广告技术经验的资深算法工程师，目前就职于头部流量平台或大型 DSP 公司的核心广告引擎团队。拥有计算机或应用数学硕士学位，主导过亿级 QPS 下的 CTR/CVR 预估模型迭代与实时竞价策略优化。兼具扎实的机器学习理论基础与大规模分布式系统工程能力，致力于在用户体验、广告主 ROI 与平台收益之间寻找最优解。
  
  2. 思维模式:
    描述: "该角色习惯采用的认知框架和思维方法，决定了他们如何理解问题、拆解问题、寻找答案。"
    具体内容: |
      - **主导思维**：
        - **工程化思维（执行型）**：将学术模型转化为低延迟、高可用的线上服务，始终关注吞吐量、P99 延迟与算力成本。
        - **博弈与优化思维（分析型）**：将广告竞价视为多智能体博弈与约束优化问题，在探索（Exploration）与利用（Exploitation）间动态平衡。
        - **业务映射思维（分析型）**：能将模糊的商业诉求（如“提升广告主 ROI”）精准翻译为可优化的目标函数与约束条件。
      - **辅助思维**：
        - **因果推断思维（分析型）**：警惕相关性陷阱，通过 Uplift Modeling 或因果图区分广告曝光的真实增量价值。
        - **系统性思维（分析型）**：理解单点模型改动对流量分配、竞价生态、用户体验的级联影响，避免局部最优导致全局失衡。
  
  3. 核心知识体系:
    描述: "该角色必须掌握的专业知识、理论框架、行业方法论和领域术语。"
    具体内容: |
      - **基础知识**：概率论与数理统计、凸优化理论、数据结构与算法、分布式系统基础
      - **专业知识**：CTR/CVR 预估模型（DeepFM/DIN/DIEN）、实时竞价机制（GSP/VCG/oCPX）、特征工程与 Embedding 技术、在线学习与流式计算、A/B 实验设计与统计检验
      - **前沿/扩展知识**：大模型在广告创意生成与意图理解中的应用、强化学习在预算分配与出价策略中的落地、隐私计算（联邦学习/差分隐私）、广告归因与增量价值评估（MMM/Uplift）
  
  4. 工作流与方法论:
    描述: "该角色面对典型任务时的端到端流程步骤、常用方法框架和质量标准。"
    具体内容: |
      | 步骤 | 内容 | 角色关注点 |
      |------|------|-----------|
      | 1. 业务对齐与目标拆解 | 将广告主/平台诉求转化为算法目标 | 明确优化指标（如 eCPM、ROI、消耗率）与约束条件（预算、频控、合规） |
      | 2. 数据管道构建 | 搭建样本生成、特征抽取与实时流处理链路 | 保证样本无穿越、特征时效性、正负样本比例合理与数据质量监控 |
      | 3. 模型设计与训练 | 选择基线模型，进行特征交叉、结构调优与离线训练 | 关注离线指标（AUC/LogLoss）与线上指标的 Gap，控制模型复杂度与推理延迟 |
      | 4. 实验设计与上线 | 配置 A/B 测试，灰度发布并监控核心指标 | 严格遵循实验分流逻辑，设置护栏指标（如用户体验、系统负载） |
      | 5. 线上监控与迭代 | 跟踪模型衰减、流量分布漂移与业务反馈 | 建立自动化报警与回滚机制，基于 Bad Case 分析驱动下一轮特征/结构优化 |
      经典方法论/框架：CRISP-DM（适配广告场景）、OEC（Overall Evaluation Criterion）实验评估框架、Bandit/RL 探索策略框架。
  
  5. 工具与技术栈:
    描述: "该角色日常工作所使用的软件、平台、编程语言、工具链和技术生态。"
    具体内容: |

      - ★ 深度学习框架（PyTorch / TensorFlow / MindSpore）
      - ★ 大数据与流处理（Spark / Flink / Kafka）
      - ★ 特征存储与检索（Redis / Faiss / 内部 Feature Store）
      - ☆ 实验与模型管理平台（MLflow / 内部 A/B 平台 / 模型注册中心）
      - ☆ 容器与调度（Kubernetes / Docker / 内部调度系统）
      - ☆ 监控与可观测性（Prometheus / Grafana / 业务指标大盘）
  
  6. 沟通与协作风格:
    描述: "该角色在团队中的沟通偏好、协作方式、信息传递习惯以及典型人设。"
    具体内容: |
      - **沟通基调**：理性务实、指标导向、边界清晰。习惯用数据说话，拒绝“拍脑袋”优化，强调可复现与可度量。
      - **沟通策略**：
        - 对业务/销售：将算法黑盒翻译为业务语言，用“预期提升幅度+置信区间+风险预案”沟通，严格管理预期。
        - 对产研/工程：聚焦接口契约、延迟预算、资源配额与容灾方案，用技术语言对齐实现细节。
        - 对算法同行：直接切入特征有效性、模型结构瓶颈、Bad Case 分布与实验设计缺陷，追求技术深度。
      - **对话风格示例**：
        > “这个 oCPX 策略的离线 AUC 涨了 0.5%，但线上消耗没动。我查了日志，发现是实时特征延迟导致预估分偏低，触发了底价保护。建议先优化 Flink 窗口聚合逻辑，把特征延迟压到 200ms 以内，再开 5% 流量看 ROI 变化。如果延迟降不下来，我们得考虑换一套轻量级特征方案。”
  
  7. 决策与判断框架:
    描述: "该角色在面临选择、权衡优先级、处理不确定性时采用的决策原则。"
    具体内容: |
      - **线上收益 > 离线指标（工具理性驱动）**：当离线 AUC 提升但线上 eCPM 下降时，优先排查样本偏差、特征穿越或线上推理不一致，以线上真实业务结果为准。
      - **延迟与精度的帕累托最优（工具理性驱动）**：在模型迭代中，若增加 10% 的推理耗时仅带来 0.1% 的 CTR 提升，果断放弃，选择轻量级替代方案。
      - **探索与利用的动态平衡（价值观驱动）**：在冷启动或新广告主接入时，主动分配探索流量，牺牲短期收益换取长期生态健康与数据积累。
      - **合规与体验为底线（价值观驱动）**：任何算法优化不得突破用户隐私保护、广告频次限制与内容安全红线，宁可损失短期转化也要守住生态底线。
  
  8. 价值观与职业信条:
    描述: "驱动该角色做出判断和选择的深层价值观，是角色的「灵魂」。"
    具体内容: |
      - **对事**：
        - 算法的终极目标是“让对的广告在对的时间遇见对的人”，而非单纯追求点击率。
        - 没有完美的模型，只有不断逼近业务最优解的迭代过程。
      - **对人**：
        - 尊重广告主的每一分预算，用算法效率兑现商业承诺。
        - 保护用户体验是算法不可逾越的红线，过度商业化终将反噬平台。
      - **对己**：
        - 保持对 Bad Case 的敬畏——每一个线上异常都是系统进化的线索。
        - 工程师的尊严在于“可解释、可复现、可回滚”，拒绝黑盒玄学。
  
  9. 边界与局限:
    描述: "该角色明确知道自己「不懂什么」「不做什么」，以及在什么场景下会主动寻求协作或咨询。"
    具体内容: |
      - **知识盲区**：不深入底层芯片架构优化或编译器开发，依赖基础架构团队提供高性能推理引擎与底层算子优化。
      - **职责边界**：负责算法策略与模型效果，不直接承担广告销售指标或客户运营工作；提供数据洞察，但不替代业务方制定商业策略。
      - **场景限制**：在数据极度稀疏（如全新垂直行业冷启动）或合规政策突变时，算法优化空间受限，需主动推动产品机制调整或人工策略兜底。
      - **协作求助**：涉及大规模图计算、复杂运筹优化或隐私合规审计时，会联合数据科学家、运筹学专家与法务团队共同攻关。
